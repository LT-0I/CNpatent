## 术语一致性规则（反"同义替换轮换"）

AI 生成的文本倾向于使用不同的同义词指代同一概念（"elegant variation"），以避免重复。**在专利中这是致命缺陷**——同一技术要素必须全文使用完全相同的术语，否则会导致权利要求范围不明、审查意见质疑。

**规则**：一旦首次使用某个术语，后续必须**逐字重复**，不得替换为同义词。

| 错误（AI式同义轮换） | 正确（专利式术语锁定） |
|---|---|
| 稀疏点云 → 初始点云 → 原始点云 | 全文统一使用"稀疏点云" |
| 体素网格 → 空间网格 → 立方体网格 | 全文统一使用"体素单元网格" |
| 损失函数 → 目标函数 → 优化指标 | 全文统一使用"损失函数" |
| 图像预处理 → 图像增强 → 前处理 | 全文统一使用"图像预处理" |

**检查方法**：生成完毕后，对每个核心技术要素 grep 全文，确认只有一种叫法。

## Formatting Rules

1. **No bold/italic in body text** — Only section titles (技术领域, 背景技术, etc.) use bold via 黑体 font
2. **No bullet lists** in 背景技术 and 发明内容 — Convert to flowing paragraphs with logical transitions
3. **No step titles** — Write "步骤S1：" inline, not as standalone heading
4. **"所述"回指的分区使用策略**（从真实专利中提取的规律）
   - **发明内容中**：严格使用"所述"回指。每个技术要素首次出现时不加"所述"，第二次及以后引用必须加"所述"
     - First mention: "通过运动恢复结构算法获取初始稀疏点云"
     - Second mention: "将所述初始稀疏点云输入..."
   - **"可选的"从属特征中**：引入句本身用"所述"回指主步骤
   - **具体实施方式中**："所述"使用频率**大幅降低**，改用更自然的回指方式："上述"、"该"、直接指代。这是发明内容与具体实施方式在行文上的最显著差异
   - **附图说明中**：不使用"所述"
5. **Semicolons between steps** in 具体实施方式: S1...；S2...；S3...。(last step ends with period)
6. **No quotation marks** around technical terms — Use them as plain text
7. **No em dash (——) overuse** — AI 倾向每段使用破折号插入补充说明，专利中应改为括号或独立句
8. **No curly quotes** — 使用直角引号「」或不加引号，不用中文弯引号""

## Section-Specific Guidelines

### 技术领域
- 1 sentence only（仅1句话，不多不少）
- Pattern A: "本发明属于[大领域]技术领域，具体涉及一种[具体方法]。"
- Pattern B: "本发明属于[大领域]技术领域，特别是涉及一种[具体方法]。"
- Pattern C: "本发明涉及[大领域]技术领域，尤其涉及一种[具体方法]。"
- 三种模式均为"先大领域，后具体方向"的双层定位结构
- 发明名称必须完整嵌入后半句，一字不改

### 背景技术
- Max 4 paragraphs（1-4段均可，根据技术复杂度自然调节）
- **段落长度应呈"漏斗式"变化**，不要所有段落等长。推荐节奏：短-短-长-短 或 中-长-短
- Paragraph 1: Clinical/engineering need (why the problem matters)
- Paragraph 2: Existing approaches and their objective limitations（可引用已有专利号，但不引学术文献）
- Paragraph 3: Why current methods fall short for this specific scenario（通常是最长段，承担核心技术分析）
- Paragraph 4 (optional): 问题收束段（若段3已足够长且末尾已自然收束，可省略独立的第4段）
- NO academic citations, NO "[1]" references（可引用专利号，如"专利号为CN...的一种..."，但不引论文）
- **转折词使用**：用"但"、"然而"、"而对于"进行层层递进转折，不用"首先...其次...最后..."
- **结尾句推荐模式**："因此，如何[技术动作]，是当前[领域名]中需要解决的技术问题。"
  - 示例："因此，如何对运动恢复结构算法在创口凹陷区域产生的点云空洞进行有效补全，并在此基础上实现高质量的三维高斯泼溅重建，是当前临床创口三维评估中需要解决的技术问题。"
  - 也可用设问式收束："如何提高[技术动作]是目前急需解决的问题。"

### 发明内容
- Pure text, NO formulas
- **必须按以下固定结构依次排列**：

#### ① 目的句（1段）
"本发明的目的是提供一种[方法名]，以解决上述现有技术存在的问题。"

#### ② 引入句（1段）
"为实现上述目的，本发明提供了一种[方法名]，包括："

#### ③ 独立权利要求主要步骤（每步独立成段）
- 每个步骤是一个独立段落，直接描述技术操作
- **不使用**"步骤S1"编号，纯文字叙述
- 步骤之间以分号（；）结尾，最后一步以句号（。）结尾
- 每步首次提及的技术要素直接引入，后续引用加"所述"
- **"问题-方案"交叉叙事（推荐）**：对于核心创新步骤，在方案段之前插入一个独立的"问题陈述段"，形成"步骤→问题A→方案A→问题B→方案B"的交叉结构：
  - 问题段：以应用对象的客观特征开头，以"导致..."结尾，描述具体技术困难
  - 方案段：以"为解决上述...问题，"/"为使..."/"为此，"开头，接技术方案
  - 问题段和方案段均以分号结尾，视为步骤列表的一部分
  - 仅在创新步骤前使用，已有技术步骤不需要问题陈述
  - 每篇专利通常有 2-4 处问题-方案对，覆盖核心创新点
  - 示例：
    ```
    肛瘘术后创口具有深层狭窄窦道结构，单目相机在此类区域的可视
    角度极度受限，运动恢复结构算法在该区域产生大面积点云空洞；
    为解决上述深层隐蔽区域的几何缺失问题，根据所述初始稀疏点云
    和相机位姿参数构建创口区域的三维空间包围盒，对所述包围盒进行
    基于立方体的空间体素剖分采样...；
    ```

#### ④ "可选的"从属特征展开（每组对应一个需要细化的主步骤）
- 引入段格式: "可选的，所述[引用主步骤中的关键技术操作原文]，具体包括："
- 后续各段: 逐步展开细节，每段独立成段
- 子步骤之间以分号（；）结尾，最后一个子步骤以句号（。）结尾
- 展开内容应比主步骤更详细，但仍为纯文字，不含公式
- 通常为 3-5 组"可选的"，覆盖核心创新步骤

#### ⑤ 技术效果（2段）
- 引入段: "本发明的技术效果为："
- 效果描述段: 以"本发明{通过/提供了}..."开头，一段话描述全部技术效果
- **推荐结构**："针对[问题A]，通过/利用[手段A]，[效果A]。针对[问题B]，利用[手段B]，[效果B]。" 逐个创新点对应一句"针对...通过..."
- 也可用分号分隔各效果写成一段
- **禁止**空泛笼统的效果描述，每条效果必须对应具体技术手段

#### 示例结构
```
[目的句]
[引入句，包括：]
  [步骤A描述]；
  [步骤B描述]；
  ...
  [步骤N描述]。
可选的，所述[步骤B关键词]，具体包括：
  [细节1]；
  [细节2]；
  [细节3]。
可选的，所述[步骤C关键词]，具体包括：
  [细节1]；
  [细节2]。
...
本发明的技术效果为：
本发明通过...；...；...。
```

### 附图说明
- **必须以通用法律声明开头**（2段），然后才是逐图描述
- 声明段1: "为了更清楚地说明本发明实施例或现有技术中的技术方案，下面将对实施例中所需要使用的附图作简单的介绍，显而易见地，下面描述中的附图仅仅是本发明的一些实施例，对于本领域普通技术人员来讲，在不付出创造性劳动的前提下，还可以根据这些附图获得其他的附图。"
- 声明段2: "构成本申请的一部分的附图用来提供对本申请的进一步理解，本申请的示意性实施例及其说明用于解释本申请，并不构成对本申请的不当限定。在附图中："
- **逐图描述固定格式**："图X为本发明实施例中{的/提供的}[描述]{示意图/流程图}。"
  - 示例：
    - "图1为本发明实施例提供的系统总体架构示意图。"
    - "图2为本发明实施例中面向肛瘘术后创口的三维重建方法整体流程示意图。"
    - "图3为本发明实施例中基于立方体的空间体素剖分采样示意图。"
  - 描述中**不加引号、不用缩写、不加解释性文字**，只写图的标题
- **逐图标点规则**：中间图以分号（；）结尾，最后一张图以句号（。）结尾（与发明内容步骤列举的标点规则一致）

### 具体实施方式
- **必须以通用法律声明开头**（5段），然后才是实施例正文
- 声明段1: "现详细说明本发明的多种示例性实施方式，该详细说明不应认为是对本发明的限制，而应理解为是对本发明的某些方面、特性和实施方案的更详细的描述。"
- 声明段2: "应理解本发明中所述的术语仅仅是为描述特别的实施方式，并非用于限制本发明。另外，对于本发明中的数值范围，应理解为还具体公开了该范围的上限和下限之间的每个中间值。在任何陈述值或陈述范围内的中间值以及任何其他陈述值或在所述范围内的中间值之间的每个较小的范围也包括在本发明内。这些较小范围的上限和下限可独立地包括或排除在范围内。"
- 声明段3: "在不背离本发明的范围或精神的情况下，可对本发明说明书的具体实施方式做多种改进和变化，这对本领域技术人员而言是显而易见的。由本发明的说明书得到的其他实施方式对技术人员而言是显而易见的。本申请说明书和实施例仅是示例性的。"
- 声明段4: "关于本文中所使用的「包含」「包括」「具有」「含有」等等，均为开放性的用语，即意指包含但不限于。"
- 声明段5: "需要说明的是，在不冲突的情况下，本申请中的实施例及实施例中的特征可以相互组合。下面将参考附图并结合实施例来详细说明本申请。"
- 声明段之后才开始实施例正文
- Engineering "how-to" style, not academic "why" explanations
- Every step must reference at least one figure
- All core computations expressed as LaTeX formulas
- Explain each variable's physical meaning after formulas
- Include concrete parameter values (e.g., N=128, K=256, λ=0.2)

## "一简一繁"描述策略

专利中同时包含已有技术（现有技术的组合使用）和本发明的创新点。二者的描述力度必须形成鲜明对比：

### 已有算法 → 弱化（一笔带过）

对 SfM、NeRF、3DGS 等已被广泛公开的通用算法，**只说"用了什么"，不展开"怎么做"**：
- 压缩为 1-2 句功能性描述，点明输入和输出即可
- 不展开其内部原理、网络结构、训练流程
- 不堆砌教科书推导，避免喧宾夺主
- 措辞："{算法名}为本领域公知技术，此处不再赘述"

示例（弱化）：
```
利用运动恢复结构算法对所述预处理后的图像序列进行特征匹配与三角化，
获得稀疏点云及各帧图像对应的相机位姿参数。
```

### 创新算法 → 强化（充分展开）

对本发明独创的改进点，必须**详尽到本领域技术人员可以复现**：
- 构建"场景痛点 → 技术选择 → 具体操作 → 公式推导 → 参数取值"的完整逻辑链
- 每一步的动机（"为什么这么做"）和操作（"怎么做"）都要写清
- 公式完整保留，变量逐一解释
- 给出具体的实施参数和取值范围

示例（强化）：
```
步骤S4中，由于肛瘘术后创口表面为柔软非刚性粘膜组织，
传统各向同性球形高斯难以贴合弯曲壁面，导致渲染边界模糊。
为此，本发明对每个高斯分布施加基于表面法向量的各向异性约束：

(a) 对补全后的致密点云中每个点 p_i，取其 K 近邻点集（K=20），
构建局部协方差矩阵并进行特征值分解...

(b) 取最小特征值对应的特征向量作为表面法向量 n_i...

(c) 通过 Gram-Schmidt 正交化，由 n_i 构建局部坐标系的旋转矩阵...
```

### 判断标准

| 描述对象 | 篇幅 | 公式 | 参数 |
|----------|------|------|------|
| 已有算法（SfM、基础NeRF、基础3DGS等） | 1-2 句 | 无或极简 | 无 |
| 创新算法（本发明的核心贡献） | 多段 + 子步骤 | 完整推导 | 具体取值 |

**核心原则**：已有算法是"工具"，一句话交代即可；创新算法是"发明点"，必须充分公开到可复现。篇幅分配上，创新部分应占具体实施方式总篇幅的 70% 以上。

## 公式分层管理策略

专利中的公式分为两类，应区别对待：

### 教科书公式（可精简）
通用算法的标准推导（如 NeRF 的位置编码、体渲染积分、3DGS 的高斯概率密度函数、alpha 混合公式等），这些公式在教科书或通用论文中已被充分公开。

处理方式：
- **不要整段删除**，而是压缩为一句概括 + 关键公式
- **领域化"蒙皮"**：将通用变量重新释义为目标领域的物理含义
- 例：NeRF 的 σ_i → "组织空间占据概率"，α_i → "次表面散射衰减因子"，c_i → "隐蔽组织颜色"

### 创新公式（必须保护）
体现本发明核心贡献的公式（如 PCA 法向量约束、Gram-Schmidt 正交化构建旋转矩阵、尺度约束、混合损失函数等）。

- 给出具体的参数取值范围

| 事实上 / 实际上 | — (删除) |
| 换言之 / 也就是说 | — (删除，或改为"即") |

---

## 句式结构检测规则（Reviewer 专用）

以下模式由 Reviewer Agent 在审查阶段检测，发现后退回 Writer 改写：

### 规则 1：平行结构检测
连续 3 个以上段落呈现相同的"主谓宾"骨架（如连续多段都以"通过...实现..."或"利用...获得..."开头），标记为"平行结构过密"，要求改写为长短交错、结构多样的句式。

### 规则 2：段落均匀度检测
连续 5 段以上的段落字数差异在 ±20% 以内，标记为"段落长度过于均匀"，要求调整为自然变化的段落长度。

### 规则 3：三段式凑数检测
效果/优点/特征恰好为 3 条（不多不少），标记为"疑似 AI 三段式"，要求据实际写——有几条写几条。

### 规则 4：连续关联词检测
连续 3 个以上段落都以关联词开头（如"因此"、"同时"、"此外"、"另外"），标记为"关联词堆砌"，要求删减多余关联词。

### 规则 5：同义词轮换检测
同一技术概念在相邻段落中使用不同表述（如"点云补全"→"点云填充"→"点云修复"），标记为"术语不一致"，统一为首次出现的术语。此规则与"术语一致性规则"配合，由 Reviewer 做全文级检查。

## 最终 DOCX 写入前自动替换（硬编码兜底）

在所有 AI 审查完成后、写入 DOCX 前，执行一轮纯规则替换作为最终兜底。此步骤不依赖 AI 判断，纯正则匹配，确保即使 AI 审查遗漏，最终文档中也不会残留高频 AI 痕迹词。

```python
import re

def final_deai_cleanup(text):
    """最终去 AI 痕迹兜底替换（纯正则，不依赖 AI 判断）"""
    replacements = [
        # 推销/夸大类
        (r'显著(?:提升|提高|改善)', '提高'),
        (r'(?:颠覆|革命|突破)性的?', ''),
        (r'(?:巧妙|精妙|独特|独创)的?地?', ''),
        (r'具有广阔的应用前景', ''),
        (r'意义深远', ''),
        (r'(?:卓越|优异|出色)的', '较高的'),
        (r'(?:完美)(?:解决|实现)', lambda m: m.group(0).replace('完美', '')),
        # AI 高频过渡词
        (r'值得注意的是[，,]?', ''),
        (r'需要指出的是[，,]?', ''),
        (r'需要强调的是[，,]?', ''),
        (r'至关重要', ''),
        (r'毋庸置疑[，,]?', ''),
        (r'不言而喻[，,]?', ''),
        (r'从本质上讲[，,]?', ''),
        (r'从根本上说[，,]?', ''),
        # 第六类：AI 高频结构模式
        (r'进一步地[，,]?', ''),
        (r'更为重要的是[，,]?', ''),
        (r'值得一提的是[，,]?', ''),
        (r'具体来说[，,]?', ''),
        (r'具体而言[，,]?', ''),
        (r'与此同时[，,]?', ''),
        (r'由此可以看出[，,]?', ''),
        (r'事实上[，,]?', ''),
        (r'实际上[，,]?', ''),
        (r'换言之[，,]?', ''),
        # 学术/论文腔（需排除法律声明中的"本文中所使用的"）
        (r'(?<!本文中所使用的.{0,10})本文(?!中所使用的)', '本发明'),
        (r'实验结果表明[，,]?', ''),
        (r'研究(?:发现|表明)[，,]?', ''),
    ]
    for pattern, repl in replacements:
        if callable(repl):
            text = re.sub(pattern, repl, text)
        else:
            text = re.sub(pattern, repl, text)
    # 清理替换后可能产生的多余空格和标点
    text = re.sub(r'[，,]{2,}', '，', text)
    text = re.sub(r'^\s*[，,]', '', text, flags=re.MULTILINE)
    return text
```

**使用时机**：在 Phase 3（整合与 DOCX 写入）中，将全文交给 `final_deai_cleanup()` 处理后再写入 DOCX。
