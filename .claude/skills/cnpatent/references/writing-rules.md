## ⚠️ v1.4 硬约束（2026-04-17 新增，最高优先级）

**所有 Writer / Reviewer / Phase 3 humanizer 闸门**必须遵循：[`v14_hard_constraints.md`](v14_hard_constraints.md)

## ⚠️ 写作主旨与篇幅（必读 - 优先级最高）

本节内容是写作整篇交底书的**主轴**，所有 Writer Agent 在动笔前**必须**理解。三个要点：(1) 四段式主旨；(2) 篇幅参考；(3) 编号符号。

### 1. 写作主旨结构（四段式 - 全文一以贯之）

整篇交底书必须围绕同一条**四段式逻辑链**展开，三、四、六各节都是这条链的不同侧面：

```
针对什么问题  →  采取什么方法  →  基于什么原理  →  解决什么问题 / 带来什么提升
   (背景技术)        (技术解决方案)       (具体实施方式)         (技术效果)
```

四个要素的对应关系：

| 要素 | 出现位置 | 写作要点 |
|---|---|---|
| **针对什么问题** | 三、背景技术 → 编号局限(1)-(N)<br>四·发明目的（开头总起句） | 必须是**目标领域工程场景下**的客观技术痛点，不是学术 benchmark 指标。每个局限独占一段。 |
| **采取什么方法** | 四·技术解决方案 → 编号步骤(1)-(N)<br>四·发明目的（中段优势条目，"通过...实现..."） | 编号步骤数 ≥ 优势条目数。每个步骤先说"做什么"，再说"具体做法"。 |
| **基于什么原理** | 六、具体实施方式（详细步骤+公式+变量解释）<br>四·技术解决方案（每个步骤末尾的"使得..."机理说明） | 公式仅在六节出现，技术解决方案只描述策略不写公式。 |
| **解决什么问题/带来什么提升** | 四·技术效果 → 编号效果(1)-(N) | 与发明目的的优势条目**一一对应、核心内容相同、措辞角度不同**。 |

**优势条目 ↔ 效果条目的对应规则**（重要）：
- 核心内容**相同**：优势(k)与效果(k)描述同一项改进
- 标题措辞角度**不同**：优势条目用**效果状态描写**（"Y较强"/"Y降低的同时提升了Z"），效果条目用**动作完成体**（"提升了Y"/"降低了Y"）
- 编号顺序**完全一致**：优势(1)(2)(3) ↔ 效果(1)(2)(3)
- **Why**：2026-04 人工审阅反馈表明，优势与效果标题完全相同会造成机械感。优势条目强调"本发明具备什么能力"，效果条目强调"取得了什么改善"，两者视角不同，标题应体现这一差异

**优势/效果条目标题命名规则**（来自 2026-04 人工审阅反馈）：
- **禁止以技术手段名词开头**：标题直接写效果/能力，技术手段在后文展开
  - 错误："分层退化判据提升退化检测的方向区分能力"
  - 正确（优势）："退化检测的方向区分能力较强"
  - 正确（效果）："提升了退化检测的方向区分能力"
- **优势条目标题模式**：`[效果名词短语] + 较强/降低/提升` 或 `[效果A降低的同时提升了效果B]`
- **效果条目标题模式**：`提升了/降低了/改善了 + [效果名词短语]`
- **Why**：以技术手段名词开头的标题（如"哈希体素各向异性降低..."）是把技术方案塞进标题，读起来像AI生成的长标签。标题只写效果，技术手段在后文"通过..."中展开

**优势/效果条目内部模式**：

优势条目：
```
（N）[效果状态描写]。通过[技术动作]，[机理或中间过程]，[效果展开]。
```

效果条目：
```
（N）[动作完成体]。在[场景/条件]下，相对于[基线]，[具体量化改善]。
```

### 2. 篇幅参考表（基于真实专利 REF2 实测）

| 章节 | 参考段数 | 参考字数 | Writer Agent | 备注 |
|---|---:|---:|---|---|
| 一、发明名称 | 1 | 15-30 | A | "一种...方法/系统/装置"，≤25字 |
| 二、所属技术领域 | 1 | 30-60 | A | 单句双层定位（大领域+具体方向） |
| 三、现有技术 | 6-9 | 1000-1600 | A | 概述 + 任务场景 + 编号局限(1)-(N) + 引出句 |
| 四·发明目的 | 5-7 | 500-900 | B | 总起 + 优势(1)-(N) + 综述收尾 |
| 四·技术解决方案 | 6-9 | 900-1400 | B | 开场（含图引用）+ 编号步骤(1)-(N) |
| 四·技术效果 | 5-7 | 700-1100 | B | 总起 + 效果(1)-(N) + 综述收尾 |
| 五、附图说明 | N | 50-120 | C | 每图一行，N=4-7 张图，不写声明段 |
| 六、具体实施方式 | 10-25 | 1000-3000 | C+D | 只写详细步骤 + 公式 + 变量解释，不写法律声明/概述/效果/总结/结尾 |

**全文目标总字数**: 4500-7500 字（不含发明名称）。低于 4000 字通常意味着技术细节不足；高于 8000 字通常意味着六节出现了应该放在四节的概述内容。

**字数控制要点**：
- 三、背景技术的编号局限(1)-(N)，每条 60-120 字，**不能太短**（局限要够具体）也**不能太长**（避免堆砌）。
- 四·发明目的的优势条目和四·技术效果的效果条目，每条 80-150 字。
- 四·技术解决方案的步骤条目，每条 80-200 字（创新步骤可长，支持步骤可短）。
- 六、具体实施方式的步骤可达 200-400 字，并穿插公式段。

### 3. 标点符号与编号约定

#### 3a. 正文标点全角（中文标点）

正文中所有标点**必须使用中文全角标点**，不得混用半角（英文）标点。

| 全角（正确） | 半角（禁止） | 说明 |
|---|---|---|
| ，。；：！？ | , . ; : ! ? | 逗号 / 句号 / 分号等 |
| （） | () | 圆括号 |
| "  " '  ' | " " ' ' | 引号 |
| —— | -- | 破折号 |

**例外**：`$...$` 内的 LaTeX 公式和 Python 代码块中使用 ASCII 标点，不受此约束。

**Why**：专利文书是中文法律文件，半角标点不符合 CNIPA 提交格式要求，且混用会导致 docx 排版对齐错位。Reviewer rubric A8 和 Phase 3 的 `deai_cleanup` 都会做机械性修补，但 Writer 在写作时就应主动避免。

#### 3b. 编号符号全角

整篇交底书的所有编号**必须使用全角圆括号**：`（1）（2）（3）`，**不使用**半角 `(1)(2)(3)`。

适用范围：
- 三、背景技术的编号局限
- 四·发明目的的编号优势
- 四·技术解决方案的编号步骤
- 四·技术效果的编号效果
- 六、具体实施方式的编号步骤
- 六、具体实施方式的公式编号（编号在 `$` 外部，如 `$r_i = ...$　　（1）`）

## 术语一致性规则（反"同义替换轮换"）

AI 生成的文本倾向于使用不同的同义词指代同一概念（"elegant variation"），以避免重复。**在专利中这是致命缺陷**——同一技术要素必须全文使用完全相同的术语，否则会导致权利要求范围不明、审查意见质疑。

**规则**：一旦首次使用某个术语，后续必须**逐字重复**，不得替换为同义词。

| 错误（AI式同义轮换） | 正确（专利式术语锁定） |
|---|---|
| 稀疏点云 → 初始点云 → 原始点云 | 全文统一使用"稀疏点云" |
| 体素网格 → 空间网格 → 立方体网格 | 全文统一使用"体素单元网格" |
| 损失函数 → 目标函数 → 优化指标 | 全文统一使用"损失函数" |
| 图像预处理 → 图像增强 → 前处理 | 全文统一使用"图像预处理" |

**检查方法**：生成完毕后，对每个核心技术要素 grep 全文，确认只有一种叫法。

## Formatting Rules

1. **No bold/italic in body text** — Only section titles (技术领域, 背景技术, etc.) use bold via 黑体 font
2. **No bullet lists** in 背景技术 and 发明内容 — Convert to flowing paragraphs with logical transitions
3. **No step titles** — Write "步骤S1：" inline, not as standalone heading
4. **禁止使用"所述"回指**
   - 全文不使用"所述"，包括发明内容和具体实施方式
   - 使用更自然的回指方式："上述"、"该"、"前述"、或直接指代
   - 首次出现的技术要素直接写全称，后续引用时用"上述X"、"该X"等
   - **Why**：实际专利代理人审阅反馈表明，"所述"的高频重复是最显眼的AI写作痕迹之一。技术交底书面向专利代理人，不是法律文件本身（权利要求书才需要"所述"），代理人会在撰写权利要求书时自行添加"所述"回指
5. **Semicolons between steps** in 具体实施方式: S1...；S2...；S3...。(last step ends with period)
6. **No quotation marks** around technical terms — Use them as plain text
7. **No em dash (——) overuse** — AI 倾向每段使用破折号插入补充说明，专利中应改为括号或独立句
8. **No curly quotes** — 使用直角引号「」或不加引号，不用中文弯引号""

## Section-Specific Guidelines

### 技术领域
- 1 sentence only（仅1句话，不多不少）
- Pattern A: "本发明属于[大领域]技术领域，具体涉及一种[具体方法]。"
- Pattern B: "本发明属于[大领域]技术领域，特别是涉及一种[具体方法]。"
- Pattern C: "本发明涉及[大领域]技术领域，尤其涉及一种[具体方法]。"
- 三种模式均为"先大领域，后具体方向"的双层定位结构
- 发明名称必须完整嵌入后半句，一字不改

### 背景技术
- **段落结构**：概述现有技术（1-2段）→ 中间层邻近技术路线段（1段）→ 编号局限性（每段一个核心局限点）→ 问题收束句
- **局限性段落必须编号**：以（1）（2）（3）...标注，每段开头用一句话总结核心局限点，再展开详细阐述
- **段落长度应呈"漏斗式"变化**，不要所有段落等长
- NO academic citations, NO "[1]" references（可引用专利号，如"专利号为CN...的一种..."，但不引论文）
- **转折词使用**：用"但"、"然而"、"而对于"进行层层递进转折，不用"首先...其次...最后..."
- **结尾句推荐模式**："因此，如何[技术动作]，是当前[领域名]中需要解决的技术问题。"
  - 示例："因此，如何对运动恢复结构算法在创口凹陷区域产生的点云空洞进行有效补全，并在此基础上实现高质量的三维高斯泼溅重建，是当前临床创口三维评估中需要解决的技术问题。"

#### 背景规则 B1：中间层邻近技术路线段（来自 2026-04-17 SLAM专利终版对比）

概述最接近现有技术（closest-baseline，如 FAST-LIO2、Point-LIO）之后、编号局限性之前，**必须插入一段覆盖"邻近但非本发明"的相关技术路线**，分向罗列这些路线及其与本发明的结构性差异。该段落不是对 closest-baseline 的重复，而是把本发明周边的"可能被审查员检索到的邻居"显式交待清楚。

| 结构 | 内容要点 |
|---|---|
| 引导句 | "在[子方向 A]方面，已有...的思路；在[子方向 B]方面，已有...的方法，如..." |
| 差异陈述 | "但前者...未利用/未引入...；后者...仅...并未..." |

示例（SLAM 专利终版）：
```
在退化检测方向化方面，已有基于雅可比矩阵奇异值分解做方向分解的思路，
以及通过子空间分析识别退化方向的思路；在体素结构方面，已有基于八叉树或
按局部点密度自适应调整体素尺度的方法，如自适应体素地图。但前者的方向
分解仍以单次扫描雅可比为输入，未利用非重复扫描激光雷达在时间窗口内的
累积覆盖特性；后者的自适应仅在各向同性基础上调整整体尺度或做均匀细分，
并未引入按环境主轴方向对体素三维边长做非均匀缩放的机制。
```

**Why**：仅列 closest-baseline 会让审查员/代理人误以为本发明周边没有其它近似技术路线；一旦检索到"方向分解 + 自适应体素"类现有技术，交底书将缺乏主动区分。显式交待邻近路线并说明其结构性局限，给后续"突出贡献"与"三步法（最接近现有技术 → 区别特征 → 技术效果）"答复提供支撑。

### 发明内容

发明内容写作规则参见文件开头的「⚠️ 写作主旨与篇幅」章节，特别是其中的「优势 ↔ 效果一一对应」「编号符号全角」与「篇幅参考表」。

- Pure text, NO formulas
- **发明目的开头句**：不要说"解决上述问题"，应说"改进上述问题"或"针对上述问题提供性能更好的方案"。"解决"暗示完美解决，实际上是"改进"
- **优势条目标题要简短**：给出直观的优势即可，详细解释在后文展开。不要把整个技术方案塞进标题句
- **技术效果条目同理**：标题短，描述另起。不要一句话写完所有内容
- **技术解决方案的步骤标题格式**：标题以句号结尾，后文另起句子描述。如"（1）时域累积扫描的主轴方向估计。在非重复扫描激光雷达的每一个时间窗口内..."
- **技术解决方案的步骤引导词**：不写"为此，"引导词，直接写方法

#### 发明内容规则 M1：发明目的必须显式标注"主创新点 vs 配套改进"（来自 2026-04-17 SLAM专利终版对比）

发明目的的首段（"本发明的目的是...以改进上述问题"）之后、优势条目（"本发明的优势体现在："）之前，**必须插入一段显式指认"主创新点"的单句/双句标注**，并把其余改进点标为"配套改进"。

结构模板：
```
本发明的主创新点为[X]（一行）。[Y1] 与 [Y2] 为主创新点的配套改进（一行）。
```

示例（SLAM 专利终版）：
```
本发明的主创新点为哈希体素地图的各向异性非均匀缩放结构。
分层退化判据与类级别动态权重为主创新点的配套改进。
```

**Why**：多步骤技术方案容易被误解为"多个并列创新"，导致单一性（unity of invention）缺陷风险上升；显式点名 1 个主创新点 + N 个配套改进，为代理人拟独立权项锁定核心创新特征、拟从属权项覆盖配套特征提供清晰依据。"主创新点"对应 CNIPA 三步法中的"区别技术特征"主干；配套改进对应主干周围的实施细节。

#### 发明内容规则 M2：技术方案步骤必须分级（核心创新步骤 vs 优选工程实施方式）（来自 2026-04-17 SLAM专利终版对比）

**技术解决方案的编号步骤只能是核心创新步骤**。平台部署、硬件调度、资源绑定等"工程实施细节"不得占据独立编号步骤位，应以"作为一种优选的工程实施方式，上述步骤(X)至(Y)可...以..."的单段方式挂接在最后一个核心步骤之后。

| 错误（AI 初稿）：把硬件调度升格为独立编号步骤 | 正确（终版）：降级为优选工程实施方式 |
|---|---|
| （1）时域累积主轴估计。<br>（2）分层退化判据计算。<br>（3）三类特征动态权重调整。<br>（4）哈希体素非均匀缩放。<br>**（5）ARM 大小核相位同步调度。** | （1）时域累积主轴估计。<br>（2）分层退化判据计算。<br>（3）三类特征动态权重调整。<br>（4）哈希体素非均匀缩放。<br><br>**作为一种优选的工程实施方式，上述步骤(1)至(4)可在 ARM 大小核异构平台上按扫描时间窗口划分为前后两个相位执行...** |

**判断标准**：若一个"步骤"具备以下两条中任一条，就属于工程实施方式，应降级：
1. 该步骤被替换（如改为同构单核平台）**不影响方法的发明本质**
2. 该步骤主要描述硬件绑定、资源调度、存储介质选择、部署拓扑

**具体实施方式中的处理**：在 六、具体实施方式 中，工程实施方式仍可作为独立编号子步骤（如 "（5）扫描时相与大小核流水线的相位同步调度"）充分公开，无需降级；降级仅针对四、技术解决方案中的编号步骤列。

**Why**：技术方案列表中的每一个编号步骤都会被代理人映射为独立权项技术特征；纳入工程调度类步骤会让独立权项捆绑硬件平台，缩小保护范围；同时引发"该技术特征是否属于必要技术特征"质疑。把工程部署降级为"优选"，既保留了充分公开，又让独立权项聚焦核心创新。

### 附图说明
- **不写法律声明段**，直接开始逐图描述
- **逐图描述格式简洁**，不加"为"、不加"本发明实施例中"等冗余修饰，直接写图的内容：
  - 格式："图X [简洁标题]{示意图/流程图}；"
  - 示例：
    - "图1 城市隧道无人机激光惯性里程计优化方法的整体系统架构示意图；"
    - "图2 时域累积扫描与主轴方向估计的流程图；"
    - "图3 沿主轴和垂直主轴分层退化判据计算的示意图；"
  - 描述中**不加引号、不用缩写、不加解释性文字**，只写图的简洁标题
- **Why 不写声明段**：技术交底书面向专利代理人，声明段是说明书的格式要求，交底书不需要。代理人撰写说明书时会自行补充
- **逐图标点规则**：中间图以分号（；）结尾，最后一张图以句号（。）结尾

### 具体实施方式
- **不写法律声明段、不写实施例标题、不写概述段、不写环境说明段**，直接从详细步骤开始
- **不写技术效果段、总结段、结尾段**，步骤写完即结束
- **Why 不写这些**：技术交底书面向专利代理人，这些法律套话和结构性段落是说明书的格式要求，交底书不需要。代理人撰写说明书时会自行补充法律声明、实施例结构、效果总结等内容
- 结合专利配图说明方法即可，已有的通识性技术不做过多阐述
- Engineering "how-to" style, not academic "why" explanations
- Every step must reference at least one figure
- All core computations expressed as LaTeX formulas
- Explain each variable's physical meaning after formulas
- Include concrete parameter values as ranges (e.g., K 取值范围 4-16；典型取值 0.3-0.5 米)
- **短句优先**：步骤描述用短句，避免一个句子包含多个从句。长句拆成多个短句，增强可读性
- **禁止方法独特性论证**：只需说清楚"方法是什么"和"怎么做"，不要用整段来论证"本方法与其他方法有什么本质区别"。区别于其他方法的特征在方法描述中顺带提及即可，不需要独立段落进行对比论证

#### 实施规则 D1：步骤体内禁止嵌入效果论证与量化结论（来自 2026-04-17 SLAM专利终版对比）

具体实施方式的步骤正文**只描述操作**（输入→动作→输出），不得嵌入"由此节省 X"、"从而避免 Y"、"满足 Z 约束"、"端到端时延为 X 毫秒"等效果陈述或量化推断。所有效果/收益/量化成绩都已在 四、技术效果 集中描述，步骤体重复会引入 AI 写作的"边叙述边邀功"风格。

| 错误（AI 初稿）：步骤体嵌入效果/推断 | 正确（终版）：只留操作 |
|---|---|
| 若已预先给定车道中心线方向先验，则用该先验向量替代上述 n_axis 并跳过在线主成分分析计算，**由此节省对应的计算耗时，同时避免因累积点云中动态目标残留而造成的主轴估计偏差**。 | 若已给定了建筑信息模型提供的车道中心线方向先验，则用该先验向量替代上述 n_axis 并跳过本子步骤的在线主成分分析计算。 |
| 前相位占 70% 时段、后相位占 30% 时段...**由于前相位与扫描过程并行执行，端到端时延仅由后相位决定，在 100 毫秒扫描窗口下为 30 毫秒，满足控制环路的时延约束**。 | 前相位占据扫描窗口时长的前 70% 时段，后相位占据后 30% 时段。相位边界由扫描窗口起止点时间戳直接触发... |

**例外保留**：某些负反例对比（"若采用固定权重，则..."、"若采用各向同性体素，则..."）是为了解释"为什么要做这一步"的必要动机段，位于子步骤之间的独立段落，可以保留。关键区别：
- **禁止**：在动作叙述句内嵌入"由此..."、"从而..."、"满足..."
- **允许**：独立段落中的负反例对比（"若不做分层处理，单一指标无法区分退化方向..."）

**Why**：效果论证散落在每个步骤会冲淡技术效果节的集中陈述；量化推断（"在 100 毫秒窗口下为 30 毫秒"）属于预期性能声明，若未经实验支撑会被审查员挑战，且与 技术效果 节的量化陈述重复容易自相矛盾。步骤体聚焦操作可复现性，效果统一在 技术效果 节中一次性声明。

## "一简一繁"描述策略

专利中同时包含已有技术（现有技术的组合使用）和本发明的创新点。二者的描述力度必须形成鲜明对比：

### 已有算法 → 弱化（一笔带过）

对 SfM、NeRF、3DGS 等已被广泛公开的通用算法，**只说"用了什么"，不展开"怎么做"**：
- 压缩为 1-2 句功能性描述，点明输入和输出即可
- 不展开其内部原理、网络结构、训练流程
- 不堆砌教科书推导，避免喧宾夺主
- 措辞："{算法名}为本领域公知技术，此处不再赘述"

示例（弱化）：
```
利用运动恢复结构算法对所述预处理后的图像序列进行特征匹配与三角化，
获得稀疏点云及各帧图像对应的相机位姿参数。
```

### 创新算法 → 强化（充分展开）

对本发明独创的改进点，必须**详尽到本领域技术人员可以复现**：
- 构建"场景痛点 → 技术选择 → 具体操作 → 公式推导 → 参数取值"的完整逻辑链
- 每一步的动机（"为什么这么做"）和操作（"怎么做"）都要写清
- 公式完整保留，变量逐一解释
- 给出具体的实施参数和取值范围

示例（强化）：
```
步骤S4中，由于肛瘘术后创口表面为柔软非刚性粘膜组织，
传统各向同性球形高斯难以贴合弯曲壁面，导致渲染边界模糊。
为此，本发明对每个高斯分布施加基于表面法向量的各向异性约束：

(a) 对补全后的致密点云中每个点 p_i，取其 K 近邻点集（K=20），
构建局部协方差矩阵并进行特征值分解...

(b) 取最小特征值对应的特征向量作为表面法向量 n_i...

(c) 通过 Gram-Schmidt 正交化，由 n_i 构建局部坐标系的旋转矩阵...
```

### 判断标准

| 描述对象 | 篇幅 | 公式 | 参数 |
|----------|------|------|------|
| 已有算法（SfM、基础NeRF、基础3DGS等） | 1-2 句 | 无或极简 | 无 |
| 创新算法（本发明的核心贡献） | 多段 + 子步骤 | 完整推导 | 具体取值 |

**核心原则**：已有算法是"工具"，一句话交代即可；创新算法是"发明点"，必须充分公开到可复现。篇幅分配上，创新部分应占具体实施方式总篇幅的 70% 以上。

## 公式分层管理策略

专利中的公式分为两类，应区别对待：

### 教科书公式（可精简）
通用算法的标准推导（如 NeRF 的位置编码、体渲染积分、3DGS 的高斯概率密度函数、alpha 混合公式等），这些公式在教科书或通用论文中已被充分公开。

处理方式：
- **不要整段删除**，而是压缩为一句概括 + 关键公式
- **领域化"蒙皮"**：将通用变量重新释义为目标领域的物理含义
- 例：NeRF 的 σ_i → "组织空间占据概率"，α_i → "次表面散射衰减因子"，c_i → "隐蔽组织颜色"

### 创新公式（必须保护）
体现本发明核心贡献的公式（如 PCA 法向量约束、Gram-Schmidt 正交化构建旋转矩阵、尺度约束、混合损失函数等）。

处理方式：
- **完整保留**，一个变量都不能省
- 公式后紧跟**每个变量的物理含义**解释
- 给出具体的参数取值范围

### 领域化"蒙皮"示例

| 通用变量 | 领域化释义（医学三维重建） | 领域化释义（工业检测） |
|----------|--------------------------|----------------------|
| σ (density) | 组织空间占据概率 | 缺陷区域占据概率 |
| α (opacity) | 次表面散射衰减因子 | 表面反射衰减因子 |
| c (color) | 隐蔽组织颜色 | 缺陷区域色彩响应 |
| T (transmittance) | 光线穿透组织的累积透射率 | 光线穿透材料的累积透射率 |

## 具体实施方式的段落结构

### 步骤命名约定

三篇样本中使用两种风格，**同一篇专利内必须统一**：

| 风格 | 格式 | 示例 |
|------|------|------|
| 编号式（推荐） | （X）标题 | （1）单目相机多角度图像采集与预处理 |
| 步骤式 | 步骤SX：标题 | 步骤S1：传感器数据接入与硬件时间同步 |

- 编号式适合步骤较多（≥6步）的方案，排版更紧凑
- 步骤式适合步骤较少（3-5步）的方案
- **标题直接写在编号后面，不另起一行**，后续内容紧跟在同一段落或下一段

### 步骤标题的命名风格

步骤标题必须使用**功能名词短语**（"什么+什么"），不用动宾短语（"做什么"）：

| 错误（动宾短语） | 正确（功能名词短语） |
|---|---|
| 采集并预处理图像 | 单目相机多角度图像采集与预处理 |
| 获取相机位姿和稀疏点云 | 运动恢复结构算法获取相机位姿与稀疏点云 |
| 补全点云空洞 | 基于立方体的神经辐射场点云补全 |
| 检测退化并选帧 | 算力-几何-光照三重感知判据与视觉帧筛选 |

命名规则：`[限定方法/约束] + [核心操作名词] + [可选：与/及 + 附加操作名词]`

**步骤标题介词多样化**（来自 2026-04 人工审阅反馈）：
当多个步骤标题以"根据X，做Y"格式命名时，不要全部使用"根据"，应在"基于"/"依据"/"根据"之间交替，避免单调重复。
- 错误：(1)根据A... (2)根据B... (3)根据C...
- 正确：(1)基于A... (2)依据B... (3)根据C...

### 子算法引入句模板

每个详细步骤的正文开头，推荐使用固定的引入句模板过渡到具体操作：

```
（3）基于立方体的空间体素剖分与神经辐射场点云补全

针对上述窦道深层盲区的结构性空洞，本发明引入基于立方体的
神经辐射场网络。具体操作步骤如下：
```

模板结构：`[1-2句动机/目的] + "具体操作步骤如下："`（或"具体包括以下步骤："）

对于引用附图的步骤，也可用：`如图X所示，基于[方法名]的[操作名]步骤如下：`

### 子步骤拆分 (a/b/c/d/e)

当一个步骤包含多个操作时，避免写成一整段，应拆分为：
- **1-2 句简短引言**（说明本步骤目的，使用上述子算法引入句模板）
- **带编号的子步骤** (a)、(b)、(c)...（每个子步骤 1-3 句）

示例：
```
（3）基于立方体的空间体素剖分与神经辐射场点云补全

针对上述窦道深层盲区的结构性空洞，本发明引入基于立方体的
神经辐射场网络。具体操作步骤如下：

a）空间包围盒归一化与体素剖分：如图4所示，根据稀疏点云
的空间分布范围构建三维包围盒...

b）神经辐射场网络预测：如图5所示，该网络采用立方体结构
约束，每个立方体单元维护独立的局部特征向量...

c）面向深层窦道的体渲染与网络训练：利用相机位姿参数，
沿每条相机射线对采样点进行体渲染积分...

d）密度筛选与点云合并：训练完成后，保留体密度超过阈值的
查询点作为补全点（取值范围为10~50，本实施例取30）...
```

注意子步骤编号格式：`a）`（小写字母+右半圆括号），不用 `(a)` 或 `A.`。

**子步骤单一职责原则**（来自 2026-04 人工审阅反馈）：
一个子步骤应只覆盖一个逻辑主题。当一个子步骤涉及 3 个以上独立操作（如滤波器重排 + 地图序列化 + 符号规范化 + 反序列化加载），应拆分为多个子步骤，每个子步骤有独立小标题。

| 错误（一个子步骤塞 4 件事） | 正确（拆为 3 个子步骤） |
|---|---|
| b）下采样滤波器叶片尺寸同步重排与长期地图序列化。[滤波器重排内容]...[序列化内容]...[符号规范化内容]...[反序列化内容]... | b）下采样滤波器叶片尺寸同步重排。[滤波器重排]<br>c）长期地图序列化与主轴符号规范化。[序列化+规范化]<br>d）历史地图加载与反变换。[反序列化] |

**Why**：单一子步骤覆盖多个独立操作时，段落过长（超过 10 行），读者无法快速定位特定操作。拆分后每个子步骤 3-6 行，便于审查和引用。

好处：
- 审查员快速定位每个操作的技术细节
- 避免整段大块文本导致关键操作被淹没
- 便于专利代理人后续撰写从属权利要求时直接引用子步骤编号

## 摘要 (Abstract) 精确模板

摘要 ≤300字，结构固定为三段式（写成一段）：

```
本发明属于[大领域]技术领域，{并公开了/公开了}一种[方法名]。
{包括/该方法}：[按步骤简要描述技术方案，每步一句，步骤间以分号分隔]。
本发明[核心技术效果1]，[核心技术效果2]。
```

**规则**：
- 摘要是独立权利要求1的压缩版，技术步骤必须与发明内容中的主步骤一一对应
- 每个步骤压缩为一句话（去掉所有细节和"所述"修饰）
- 技术效果压缩为1-2句，只写最核心的效果
- 不使用"可选的"展开，不写从属特征
- 不出现"其特征在于"

```
规划着降路径。本发明最大程度上规避了自然背景光的影响、昼夜通用
且位姿测量精度更高。
```

## 发明内容中的"动机句"模式

在发明内容的步骤段落中，**创新步骤**前可以插入一段"动机句"，说明该步骤要解决的具体技术问题。这使方案的技术逻辑更清晰，也为后续审查意见答复提供支撑。

**结构**：`[问题描述段] + [解决方案段]`

**问题描述段**以技术要素的客观特征开头，以"导致...问题"结尾：
```
肛瘘术后创口具有深层狭窄窦道结构，单目相机在此类区域的可视
角度极度受限，窦道内部光线难以到达且表面纹理匮乏，运动恢复
结构算法在该区域产生大面积点云空洞，传统基于特征匹配的稠密化
方法在无光、无特征的深层盲区中无法获取有效匹配信息；
```

**解决方案段**以"为解决上述...问题，"或"为此，"开头，接技术方案：
```
为解决上述深层隐蔽区域的几何缺失问题，根据所述初始稀疏点云和
相机位姿参数构建创口区域的三维空间包围盒，对所述包围盒进行
基于立方体的空间体素剖分采样...
```

**使用规则**：
- 仅在**创新步骤**前使用动机句，已有技术步骤不需要
- 动机句应描述**客观技术困难**，不要写成推销语言
- 每篇专利的发明内容中通常有 2-4 处动机句，覆盖核心创新点

## 公式编号与引用约定

具体实施方式中的独立公式必须编号，编号从（1）开始连续递增。

### 公式段落样式
所有数学公式段必须使用模板的 `公式` 段落样式（通过 `add_formula(doc, latex_text)` 写入），并以纯文本 LaTeX 字符串呈现。Python 端的 LaTeX 字符串必须使用原始字符串字面量 `r'...'`，否则 `\a` / `\b` / `\f` 会被解释为控制字符，导致 docx 写入失败。

### 公式编号格式
独立公式（单独成段）左右各用一个 `$` 包裹，编号在 `$` **外部**，用全角括号：
```
$r_i = \mathbf{n}_i^T (\mathbf{T} \cdot \mathbf{p}_i^L) - d_i$　　（1）
```

**注意**：不使用 `$$...$$` 双美元符号。行内公式（嵌入正文句子中）同样用单 `$` 包裹，如"其中 $T_i$ 为累积透射率"。独立公式与行内公式的区别仅在于是否单独成段，`$` 包裹方式相同。

### 公式引用格式
正文中引用公式时使用"如式（X）"或"式（X）"：
```
根据上述得到的7个点坐标，最优化如式（2）的圆方程的目标函数...
利用式（3）将7个紫外光点的像素坐标集合转为以圆心坐标为原点的极坐标形式。
```

### 变量释义格式
公式后紧跟变量释义段，用"其中，"引导：
```
其中，$T_i$ 为累积透射率，$\alpha_i$ 为第 $i$ 个采样点的不透明度，
计算方式为：
```

创新公式的变量必须做**领域化释义**（参见公式分层管理策略中的"蒙皮"示例）。

### 叙述式操作序列应提取为独立公式（来自 2026-04 人工审阅反馈）

当一个段落以自然语言描述一个多步数学运算（如"先做变换X，再做除法Y，再取整得到Z"）时，若该运算可以用一个简洁公式表达，应提取为独立编号公式，不要全部埋在叙述中。

| 错误（叙述式） | 正确（提取公式） |
|---|---|
| "先做坐标变换 $p' = \mathbf{S}^{-1} \cdot p$，再按基准边长 $\Delta$ 对 $p'$ 的三个分量分别取整得到索引键 （$k_1$，$k_2$，$k_3$）" | "按下式计算索引键：" + 独立公式 $(k_1, k_2, k_3) = \lfloor \mathbf{S}^{-1} \cdot p / \Delta \rfloor$　（N） |

**判断标准**：如果叙述中包含 2 个以上数学运算步骤（变换、除法、取整、求和等）且最终结果是一个可命名的输出量，就应该提取。单步赋值（如 $\hat{\mathbf{n}} = \mathbf{v}_1 / \|\mathbf{v}_1\|$）可以保留内联。

**Why**：叙述式操作序列可读性差，读者需要在长句中人肉解析运算顺序。提取为公式后运算结构一目了然，也便于本领域技术人员直接复现。

### 公式穿插的"夹叙夹议"写法（来自真实专利样本）

公式不是简单罗列，而是嵌入在解释性叙述中，形成"意图→公式→结果/用途"的三明治结构：
- 先用文字说明**这一步要做什么**和**变量含义**
- 嵌入公式
- 再用文字解释**公式的结果和用途**

对于创新公式，还可在变量释义中穿插**物理意义的直觉解读**，用破折号引出：
```
$\sigma_i$ 为第 $i$ 个采样点的体密度，表征该采样点处创口组织的
空间占据概率——在深层窦道盲区内，$\sigma_i$ 的预测值反映了网络
对该处是否存在隐蔽组织的推断置信度
```

这种"公式+物理直觉"的穿插写法，在纯数学推导中注入领域知识层面的理解，使技术方案更具说服力。注意：破折号引出的物理解读仅用于创新公式，教科书公式不需要。

### 六a、短句与可读性规则（来自 2026-04-15 人工审阅反馈）

以下规则是从人工审阅专利交底书时反复出现的修改意见中总结而来。这些问题的共同特征是"读起来像AI生成的长篇大论"，降低了交底书的可读性和专业感。

**规则 S1：长句拆短句**
一个句子超过 40 字时，考虑拆成 2-3 个短句。步骤小标题、效果条目标题尤其要短。长复合句是最显眼的AI写作特征之一。

**规则 S2：步骤小标题要简短**（补充上文"步骤标题的命名风格"章节）
步骤标题（如"（3）分层退化指标调整三类特征权重"）应是一个简短的功能名词短语，不要把整个方法都塞进标题。详细内容在后文展开即可。

**规则 S3：禁止方法独特性论证段落**
不要写"此处需明确强调：本步骤...而不是...也不是...存在本质区别"这样的整段论证。这是论文写法，不是专利交底书写法。
- 交底书只需说清楚"方法是什么、怎么做"
- 区别于其他方法的特征在方法描述中顺带点明即可
- 不需要单独段落进行对比论证

**规则 S4：禁止"直观机理"解释段落**
不要在公式后写"上式的直观机理为：当X增大时...当Y增大时..."这样的论证段落。专利交底书是阐述方法的，不是写论文做论证。必要的解释融入方法叙述中即可。

**规则 S5："X驱动的Y"句式是AI特征**
"退化判据驱动的特征权重调整"、"分层退化判据驱动的三类激光特征类级别动态权重调整"这类句式可读性很差。改为两个短句或用更自然的表述：
- 错误："主轴驱动的哈希体素几何尺寸三维边长非均匀缩放"
- 正确："根据主轴方向，对哈希体素做非均匀缩放"

**规则 S6：通识性技术简写**（补充上文"一简一繁"描述策略）
已有的、通识性的技术（如主成分分析、协方差矩阵、特征值分解等教科书内容）不要做过多阐述。公式写清楚，一句话说明用途即可。创新部分才需要详细展开。

**规则 S7：参数用范围/典型取值，禁止"本实施例取值 X"锚点**（2026-04-17 扩展至具体实施方式）

- 四、发明内容 中的参数**只给取值范围**（"取值范围为 4 至 16"），不写"本实施例取值 X"。
- 六、具体实施方式 中的方法叙述同样**只给取值范围或"典型取值 A 至 B"**，不写逐参数"本实施例取值 X"锚点。如果必须给出具体实施数值，应集中在一个独立的参数表或 实施例 数据块中，而不是散落在步骤正文里。
- **允许保留的固定值**：方法结构性的固定参数（如"前相位占扫描窗口 70%，后相位占 30%"、"绑定到 4 个小核"）是方法的组成部分，不是可调的样例值，可直接给出具体数值。
- **禁止形式**："本实施例取值 8"、"本实施例取值 16"、"本实施例中为 100 毫秒，前相位 70 毫秒、后相位 30 毫秒"。

| 错误（AI 初稿）：步骤正文散落"本实施例取值 X" | 正确（终版）：只给范围 |
|---|---|
| 沿轴切片总数的取值范围为 4 至 16，**本实施例取值 8**；垂轴扇区总数的取值范围为 8 至 32，**本实施例取值 16**。 | 沿轴切片总数 K 的取值范围为 4 至 16；垂轴扇区总数 K' 的取值范围为 8 至 32。 |
| σ 为归一化尺度参数，取值范围为 0.1 至 1.0，**本实施例取值 0.3**；ε 为防零除小量，**本实施例取值 1e-6**。 | σ 为归一化尺度参数，取值范围为 0.1 至 1.0；ε 为防零除小量。 |

**Why**：
1. "本实施例取值 X" 在方法叙述里散落 5-10 处会制造"虚假精度"，让审查员误以为这些具体值是发明本质；
2. 具体值一旦写入方法描述，代理人在拟写权利要求时可能被迫把这些值作为必要技术特征纳入从属权项，压缩了权项覆盖范围；
3. 典型取值范围已足以支撑"充分公开"要求，无需把一组特定值绑死在正文；
4. 如果审稿/评估需要看具体数值，应由代理人在 说明书 中另设"实施例一/实施例二"章节呈现，与方法描述区分开。

**规则 S8：inline 变量必须用 $ 包裹**
正文中出现的数学变量（如 d_axis、w_corner、β_1 等）都要用 $d_{\mathrm{axis}}$、$w_{\mathrm{corner}}$、$\beta_1$ 这样的 LaTeX 格式包裹。裸写变量名是格式缺陷。

### 六、AI 高频结构模式

| Forbidden (AI味) | Replacement (专利味) |
|---|---|
| 总之 / 综合来看 / 由此可见（段首） | — (删除，或改为具体技术陈述) |
| 进一步地 / 更为重要的是 | — (删除) |
| 值得一提的是 / 令人振奋的是 | — (删除) |
| 正是因为...才能够 | — (改为直接陈述因果) |
| 这一策略 / 这一方案 / 这一方法 | 上述方法 / 该方案（用"上述"或"该"回指） |
| 具体来说 / 具体而言 | — (删除，直接展开) |
| 与此同时 | — (删除，直接衔接) |
| 由此可以看出 / 可以发现 | — (删除) |
| 事实上 / 实际上 | — (删除) |
| 换言之 / 也就是说 | — (删除，或改为"即") |
| 而非X / 而不是X（综述段末尾） | — (删除否定对比分句，只保留肯定陈述) |

## 句式结构检测规则（Reviewer 专用）

以下模式由 Reviewer Agent 在审查阶段检测，发现后退回 Writer 改写：

### 规则 1：平行结构检测
连续 3 个以上段落呈现相同的"主谓宾"骨架（如连续多段都以"通过...实现..."或"利用...获得..."开头），标记为"平行结构过密"，要求改写为长短交错、结构多样的句式。

### 规则 2：段落均匀度检测
连续 5 段以上的段落字数差异在 ±20% 以内，标记为"段落长度过于均匀"，要求调整为自然变化的段落长度。

### 规则 3：三段式凑数检测
效果/优点/特征恰好为 3 条（不多不少），标记为"疑似 AI 三段式"，要求据实际写——有几条写几条。

### 规则 4：连续关联词检测
连续 3 个以上段落都以关联词开头（如"因此"、"同时"、"此外"、"另外"），标记为"关联词堆砌"，要求删减多余关联词。

### 规则 5：同义词轮换检测
同一技术概念在相邻段落中使用不同表述（如"点云补全"→"点云填充"→"点云修复"），标记为"术语不一致"，统一为首次出现的术语。此规则与"术语一致性规则"配合，由 Reviewer 做全文级检查。

## 最终 DOCX 写入前自动替换（硬编码兜底）

在所有 AI 审查完成后、写入 DOCX 前，执行一轮**纯规则替换**作为最终兜底。此步骤不依赖 AI 判断，确保即使 Reviewer 漏掉，最终文档中也不会残留高频 AI 痕迹词。

实现位于 `scripts/deai_cleanup.py`，使用方法：

```python
from deai_cleanup import final_deai_cleanup

cleaned = final_deai_cleanup(merged_text)
# 然后将 cleaned 写入 DOCX
```

`final_deai_cleanup()` 覆盖七大类禁用词：

- **一、推销/夸大类**：显著提升 / 颠覆性 / 巧妙地 / 完美解决 / ...
- **二、AI 高频过渡词**：值得注意的是 / 至关重要 / 毋庸置疑 / 为此，/ ...
- **六、AI 高频结构模式**：进一步地 / 具体来说 / 与此同时 / 事实上 / ...
- **六a、可读性问题**：驱动信号 → 调整依据 / 输入信号
- **五、学术/论文腔**：本文 / 实验结果表明 / 研究发现 / ...
- **七、"所述"回指**：全文替换"所述"为空字符串（交底书不使用"所述"）

替换后会自动清理连续逗号和行首逗号。完整替换表见 `scripts/deai_cleanup.py` 的 `REPLACEMENTS` 常量；分类与理由见本文的「Anti-AI Vocabulary Replacement Table」章节。

**使用时机**：在 SKILL.md Phase 3（整合与 DOCX 写入）中，将合并后全文交给 `final_deai_cleanup()` 处理后再写入 DOCX。

## v1.5 硬约束扩展（2026-04-17 新增，对应 v2 AI 味分析报告 16 条）

来自《专利文档AI味特征分析报告_v2》。这里新增 4 条硬约束（v14 简表条目 ⑪–⑭）以及两条章节差异化合并规则，配合 cnpatent-humanizer v1.5 脚本（`param_segment.py`、`term_pronoun.py`、`topic_switch.py`、`enumeration_check.py`、`background_leak.py`、`unprepared_concept.py`、`merge_short.py`）自动检测。

### 规则 ⑪：参数段纯度（「其中，」段禁操作动词）

以「其中，」开头的参数说明段**只允许参数解释**。每句必须形如「X 为 Y」「X 是 Y」「X 与 Y 分别为 ...」。

- **禁出现动词**：利用、将、使用、获得、由此、通过、根据、计算、执行、采用、引入、构建、优化、同步、更新、校验、标定、反投影、投影
- 一旦出现上述动词，**从该动词处断段**，后续内容归入下一段（属于操作步骤描述）
- 括号内字符数 >15 且含操作动词时，**剥离括号内容为独立段**

**反例**：
```
其中，$\mathbf{R}_{bc}$ 与 $\mathbf{t}_{bc}$ 为相机与机体之间的外参标定量
（通过离线标定获取，每次巡检启动时利用 PnP 算法在线校验...），...
```

**正例**（两段）：
```
（参数段）其中，$\mathbf{R}_{bc}$ 与 $\mathbf{t}_{bc}$ 为相机与机体之间的外参标定量；
          $\mathbf{P}_w$ 为病害在世界坐标系下的三维坐标。

（操作段）上述外参标定量通过离线标定获取。每次巡检启动时用 PnP 算法在线校验。
          旋转偏差大于 0.5° 或平移偏差大于 0.02m 时更新。
```

**Why**：参数段与操作步骤混杂会让读者反复在"定义—执行"之间切换视角，是专利 AI 稿最高密度指纹之一。参数段保持纯 declarative 句式、操作段保持 imperative 句式，两段之间用空行清楚分隔。

### 规则 ⑫：段内长术语代词化

≥6 字的专有术语在**同一段内**出现 ≥2 次时，第二次起必须替换为「该 + 简称 / 上述 + 简称 / 前述 X」等代词结构；术语锁定（不得换同义词）原则在**跨段**仍然生效。

| 术语 | 段内第二次起 |
|---|---|
| 几何先验场 | 该先验场 / 先验场 |
| 分布式反向修正 | 该修正机制 / 该反向修正 |
| 闭环双向约束 | 该闭环约束 |
| 多帧投影一致性校验 | 该一致性校验 |

**保留原则**：跨段落首次出现时必须写全称（保障术语锁定），段内二次起用代词（降重复感）。这与"术语锁定"并不矛盾——术语锁定禁止同义轮换（不得把「稀疏点云」换成「初始点云」），不禁止代词回指。

### 规则 ⑬：话题切换必分段

单段内出现 ≥2 个话题切换标志词时**必须分段**；单段 >250 字且含 ≥1 个话题切换标志词时**强制按标志词位置断段**。

**标志词清单**：
- 「架构上，」「观测上，」「稳定性上，」「操作上，」「效果上，」「性能上，」「机制上，」「原理上，」「流程上，」
- 「架构方面」「观测方面」「稳定性方面」「操作方面」「效果方面」「性能方面」「机制方面」「原理方面」「流程方面」「观测处理方面」
- 「X 层面」「X 角度」「X 维度」（X 为 2-4 字名词）

**禁止**：在**发明目的**小节中把上述词作为**子项标题前缀**使用（违反规则 ⑮）。发明目的子项必须用「（1）（2）（3）」编号 + 效果动词开头。

### 规则 ⑭：「将 X，获得 Y」翻译腔禁用

以「将」作为动作主动词、后跟「获得 / 得到 / 取得 / 产出」式结果陈述的翻译腔句式禁止。

| 反例 | 正例 |
|---|---|
| 将四角投影坐标沿 $\mathbf{n}$ 投影至病害平面后，取对边平均长度作为物理宽度。 | 四角投影坐标沿 $\mathbf{n}$ 投影至病害平面。对边平均长度为物理宽度。 |
| 将 M 个病害聚类最终三维坐标反投影至图像坐标系，获得修正参考点。 | 对 M 个病害聚类最终三维坐标做反投影至图像坐标系，得到修正参考点。 |

**重写模板**：
- 「将 X [介词] Y，获得 Z」 → 「对 X 做 [介词] Y，得到 Z」
- 单段内「将」作句首出现 ≥2 次时，第二次起替换为「对」「用」「取」

### 规则 ⑮：发明目的 / 技术效果 必须 (1)(2)(3) 编号 + 效果动词开头

修订（2026-04-17）：发明目的小节的优势条目标题**与技术效果条目一样**，必须以**效果动词**开头，取消旧规则中「优势 = 效果状态描写」与「效果 = 动作完成体」的严格对立分工。两者的**差异**改由措辞角度体现：

- **优势条目**：「（N）[效果动词] + [对象] + [机理简述]。」
  - 例："（1）提升虚警率判定能力。通过..."
- **效果条目**：「（N）[效果动词] + [对象] + [量化改善]。」
  - 例："（1）虚警率显著降低。在...场景下，相对于...基线，下降约八成。"

**允许的效果动词**：提升、增强、降低、消除、实现、支持、获得、保障、解决、改善、提高、减少、打破、简化

**禁止**：
- 用「架构方面」「观测处理方面」「稳定性方面」「独立于 X」等自由结构作为子项**标题**（这是 AI 分块方式，不是专利分点写法）
- 子项首句字数 ≤ 25 字（效果动词 + 对象 + 能力名词）

### 规则 ⑯：技术解决方案 §4b 步骤 = 标题与正文合并、禁导语、单步 ≤ 150 字

**§4b 技术解决方案**的编号步骤一律以**"（N）标题：正文"**或**"（N）标题。正文"**（同段合并）呈现。

- ❌ 禁：标题独立成段 + 正文另起段落（两段分隔，读起来像 AI 分块）
- ✅ 正确："（1）数据采集与同步：可见光相机、激光雷达与 IMU 同步采集..."

**同时**：
- 步骤正文**禁以"为解决/为此/针对...问题，"导语开头**。直接陈述操作。
- 步骤正文**禁嵌入背景问题描述**（「X 缺乏 Y」「观测稀疏、延迟、含噪」等描述属于**背景技术**，不应散落到方案步骤里）。
- 单步骤段落字数 ≤ **150 字**（§4b 专用阈值，比 v1.4 硬约束 ② 的 200 字更严）。
- 超 150 字必须按动作链断段为「（N）... - (a) ...; (b) ...; (c) ...」形式；或拆为多个编号步骤。

### 规则 ⑰：具体实施方式 §6 步骤 = 标题独立成段（保留原规则）

**§6 具体实施方式**的子步骤保持原规则："（N）步骤标题"独立成行 + 正文另起。这与 §4b 不同，因为 §6 内每个步骤含公式、参数、子步骤 a) b) c)，标题独立成段有利于视觉层级。

- §4b（技术解决方案）：**合并**（标题+正文同段）
- §6（具体实施方式）：**独立**（标题独立成行）
- **差异来源**：§4b 每步 80-200 字属于"摘要式"描述，合并紧凑；§6 每步 200-400 字属于"展开式"描述，分开便于阅读。

### 规则 ⑱：已有技术声明限频

「为本领域已有技术」/「为本领域公知技术」在全文中出现 ≤ **3** 次。同类技术（如卡尔曼滤波器、主成分分析、PCA、PnP 算法）**只声明 1 次**，后续出现时不再追加声明。

已有技术描述的单步骤长度 ≤ **60 字**（含公式）。创新步骤篇幅占 §6 总篇幅 ≥ **70%**。

### 规则 ⑲：删空洞元描述（"链路 / 相比 / 操作步骤如下"）

以下句式在 Writer 层禁用；若 Reviewer 发现残留，humanizer regex_clean 兜底删除：

- "其中步骤（X）至步骤（Y）构成 ... 链路。"（元描述，不携信息）
- "本方法相比 ... 取得以下效果。"（引言段，删）
- "具体操作步骤如下。"（过渡句，删）
- "本实施例定义 ... 种 ..."（为列表铺垫，删）
- "本步骤为本发明的核心技术创新所在。"（自夸元话语，删）
- "具体操作步骤如下。"、"按下式计算："后为空段时 → 并入下一段
