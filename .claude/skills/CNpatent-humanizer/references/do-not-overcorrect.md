# 不过度矫正清单（Don't Over-correct）

> 本文档定义在专利文本中**应当保留**的"刻板"表达。这些表达在通用 humanizer 看来是 AI 痕迹，但在专利场景中是**法定/规范要求**，必须保留。
>
> **核心判定原则**：对任何疑似 AI 痕迹，先问"这个表达在真实代理人撰写、审查通过的专利中是否常见？"。如果答案是**是**，**保留**；如果答案是**否**，**删除/重写**。

---

## 一、法定开篇套话（必须保留）

这些是国家知识产权局审查指南允许、代理人惯用的开篇语，不算 AI 套话：

| 表达 | 出现位置 |
|---|---|
| 本发明涉及 X 技术领域 | 二、技术领域 |
| 本发明属于 X 技术领域 | 二、技术领域 |
| 本发明的目的是提供一种 X | 四、发明内容·发明目的 |
| 本发明的优势体现在 | 四、发明内容·发明目的 |
| 为实现上述目的，本发明提供了一种 X，包括 | 四、发明内容·技术解决方案 |
| 本发明的技术效果为 | 四、发明内容·技术效果 |
| 本实施例中提供了一种 X | 六、具体实施方式 |
| 本实施例的具体实施方案包括 | 六、具体实施方式 |
| 本实施例的技术效果为 | 六、具体实施方式 |

**注意**：通用 humanizer 看到"本发明的目的是提供一种"这种程式化句子会
认为是模板套话，但在专利中这是**法定开场**，删了反而违规。

---

## 二、所述回指（必须保留）

通用 humanizer 看到"所述 X 包括所述 Y 和所述 Z" 会认为是冗余、机械，但
在专利中这是**权利要求清晰性的关键**——所有反复出现的技术要素必须用
"所述"回指，否则审查员会指出权利要求范围不清。

**保留规则**：
- 在"四、发明内容"中，技术要素首次提及不加"所述"，后续提及**必须**加
- 在"五、附图说明"中**禁用**"所述"
- 在"六、具体实施方式"中**降低**"所述"频率，但仍可保留关键要素的回指

**绝对不要**因为"看起来重复"就删除合法的"所述"。

---

## 三、问题-方案叙事（必须保留）

真实创新专利常用"问题陈述段 + 方案段"的交叉叙事：

```
肛瘘术后创口具有深层狭窄窦道结构，单目相机在此类区域的可视角度极度
受限，运动恢复结构算法在该区域产生大面积点云空洞；
为解决上述深层隐蔽区域的几何缺失问题，根据所述初始稀疏点云和相机
位姿参数构建创口区域的三维空间包围盒...
```

通用 humanizer 看到"为解决上述...问题"会认为是 AI 套话，但这是真实
专利中的标准过渡。**保留**这种结构。

判定：
- 如果"为解决上述...问题"前面有具体的问题描述段，**保留**
- 如果"为解决上述...问题"前面没有具体问题段，仅是凭空出现，**删除**

---

## 四、附图引用（必须保留）

通用 humanizer 可能认为"如图 X 所示"在每段开头出现是重复，但专利中**每个
步骤都应该引用至少一张图**，"如图 X 所示"是法定引用方式。

| 表达 | 是否保留 |
|---|---|
| 如图 1 所示 | 保留 |
| 如图 1 至图 3 所示 | 保留 |
| 如图 1 和图 2 所示 | 保留 |
| 参考图 1 | 改为"如图 1 所示" |
| 见图 1 | 改为"如图 1 所示" |
| 图 1 显示了 | 改为"如图 1 所示，..." |

---

## 五、公式变量释义（必须保留）

公式后的"其中，X 表示 ..., Y 表示 ..." 是数学符号定义的标准格式。
通用 humanizer 看到"其中"出现多次会标记为 AI 痕迹，但在公式后的释义段
中这是**必须的**。

**保留规则**：在公式段（含 `$...$` 或 `$$...$$`）的紧邻下一段中，
"其中，"开头**保留**，不视为 AI 套话。

---

## 六、长复合句（按场景保留）

通用 humanizer 倾向把 50+ 字的句子标记为"过长"，但专利描述复杂技术
流程时 50-80 字是必要的。

**判定阈值**：
- 通用文本：> 40 字标记
- 专利文本：> 80 字才标记
- 法律声明段：不标记（这些句子本来就长）

示例（合法长句）：
```
利用运动恢复结构算法对所述预处理后的图像序列进行特征匹配与三角化，
获得稀疏点云及各帧图像对应的相机位姿参数，并根据所述稀疏点云的空间
分布范围构建三维包围盒。
```
（74 字，专利场景中应当保留）

---

## 七、被动语态（保留）

通用 humanizer 倾向把"X 被 Y 处理"改为"Y 处理 X"。在专利中：
- "X 被 Y 处理" 是合理的（强调被处理的对象）
- 不应该一刀切把所有被动语态改为主动

判定：只有当被动语态与"由"组合形成"X 由 Y 进行处理"这种动词名词化结构
时才修改，单纯的"X 被 Y 处理"保留。

---

## 八、专业术语的"教科书式"重复（保留）

通用 humanizer 鼓励"避免重复"（elegant variation），但专利**要求重复**。
不要把"稀疏点云"改成"初始点云"或"原始点云"，无论它在文中出现多少次。

详见 [scoring-system.md](scoring-system.md) 中"术语漂移"被列为致命级。

---

## 九、合理的对冲词（按场景保留）

按场景判定对冲词的合理性：

| 场景 | 对冲合理性 | 示例 |
|---|---|---|
| 参数取值范围 | **保留** | "本实施例取 30，取值范围为 10~50" |
| 多种实施方式说明 | **保留** | "可以采用 X 方式，也可以采用 Y 方式" |
| 抽象描述 | **保留** | "可选地，所述 X 包括..." |
| 凭空对冲 | **删除** | "可能在某种程度上影响..." |
| 4-5 层对冲堆积 | **删除** | "可能、也许、大致上、在一定程度上..." |

判定：单层"可"/"可以"/"可选地"在专利中是合法的；多层堆叠才是 AI 痕迹。

---

## 十、固定结尾段（必须保留）

具体实施方式必须以下面这段固定文本结尾，**逐字保留**，不允许任何修改：

```
以上所述，仅为本申请较佳的具体实施方式，但本申请的保护范围并不局限于此，
任何熟悉本技术领域的技术人员在本申请揭露的技术范围内，可轻易想到的变化或
替换，都应涵盖在本申请的保护范围之内。因此，本申请的保护范围应该以权利要求
的保护范围为准。
```

通用 humanizer 看到"任何熟悉本技术领域的技术人员"、"可轻易想到的变化或
替换"会认为是冗长、模糊，但这是**法定声明**，必须保留。

详见 [protected-regions.md](protected-regions.md)。

## 实现：保留判定函数

```python
import re

# 法定开篇套话（不算 AI 痕迹）
LEGITIMATE_PATENT_PHRASES = [
    r'本发明涉及.{1,20}技术领域',
    r'本发明属于.{1,20}技术领域',
    r'本发明的目的是提供一种',
    r'本发明的优势体现在',
    r'为实现上述目的，本发明提供了一种.{1,30}，包括',
    r'本发明的技术效果为',
    r'本实施例中提供了一种',
    r'本实施例的技术效果为',
    r'为解决上述.{1,20}问题',
    r'如图\s*\d+(?:\s*[至和]\s*图\s*\d+)?\s*所示',
    r'综上所述，本实施例',  # 注意这个特殊：在结尾段是合法的
]

# 法定结尾段
LEGAL_CLOSING = r'^以上所述，仅为本申请较佳的具体实施方式'


def is_legitimate_patent_phrase(text):
    """检查文本是否包含合法的专利套话"""
    for pattern in LEGITIMATE_PATENT_PHRASES:
        if re.search(pattern, text):
            return True
    if re.match(LEGAL_CLOSING, text):
        return True
    return False


def filter_overcorrect(detected_issues, paragraphs):
    """从 Phase 1 检测结果中过滤掉合法的专利套话"""
    filtered = []
    for issue in detected_issues:
        para_idx = issue.get('paragraph')
        if para_idx is not None:
            para_text = paragraphs[para_idx]
            if is_legitimate_patent_phrase(para_text):
                # 这是合法套话，跳过
                continue
        filtered.append(issue)
    return filtered
```

## 调试：判断"过度矫正"

如果发现 Phase 3 重写后的文本读起来"不像专利"了，通常是过度矫正。
检查：
1. "本发明的目的是提供" 是否被改成了"本发明提供"或"我们提出"
2. "所述"是否被全文清除
3. "为实现上述目的，本发明提供了一种" 是否被精简成"本发明提供了"
4. 法律声明段是否被改写
5. "如图 X 所示"是否被替换成"参考图 X"

若有以上情况，立即回滚并将相关 phrase 加入本文档的保留清单。
