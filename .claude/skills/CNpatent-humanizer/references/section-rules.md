# 各章节去 AI 味的差异化规则

> 中国专利交底书的各个章节有不同的语言规范。本文档描述每节的特殊规则，
> CNpatent-humanizer 在执行 Pass D 时应根据当前段落所在节，应用对应规则。

---

## 一、发明/实用新型名称

**字数**：≤25 字

**禁止**：
- 副词修饰（"一种高效的..." → "一种..."）
- 应用前景描述（"一种适用于...场景的..." → 仅用方法本质描述）
- 学术风格命名（"基于 Transformer 的..." → "基于注意力机制的..."）

**结构**：必须以"一种"开头，以"方法"/"系统"/"装置"结尾。

**改写示例**：
| AI 写法 | 专利写法 |
|---|---|
| 高效准确的图像三维重建方法 | 一种基于神经辐射场的图像三维重建方法 |
| 一种新颖的目标检测系统 | 一种基于注意力机制的目标检测系统 |
| 创新性的故障诊断装置 | 一种基于多传感器融合的故障诊断装置 |

---

## 二、技术领域

**字数**：1 句话

**固定句式**：
- "本发明属于 [大领域] 技术领域，具体涉及一种 [具体方法]。"
- "本发明属于 [大领域] 技术领域，特别是涉及一种 [具体方法]。"
- "本发明涉及 [大领域] 技术领域，尤其涉及一种 [具体方法]。"

**禁止**：
- 多于一句话
- 解释性内容（"该领域近年来发展迅速..." → 删除）
- 应用场景描述（"广泛应用于..." → 删除）

**改写规则**：若发现技术领域节超过 1 句，删除多余句子。

---

## 三、背景技术

**段数**：1-4 段

**段落长度**：呈漏斗式变化，禁止段落长度均匀

**禁用**：
- 学术引文格式（`[1]`、`(Smith, 2020)` → 删除）
- 论文文献引用（"如 Wang 等人提出..." → 删除）
- 实验数据描述（"实验结果表明..." → 删除）
- 套话开场（"随着技术的发展..." → 删除）

**允许**：
- 引用专利号（"专利号为 CN... 的..."）
- 用"但/然而/而对于"做转折，不用"首先...其次..."

**结尾固定模式**：
```
因此，如何 [技术动作]，是当前 [领域名] 中需要解决的技术问题。
```

或设问式：
```
如何 [技术动作] 是目前急需解决的问题。
```

**Pass D 检测项**：
1. 段数超过 4 → 合并相邻段
2. 段落长度方差小于均值 × 0.15 → 标记
3. 出现学术引文格式 → 删除
4. 出现"首先/其次/最后" → 删除并改写
5. 结尾不符合固定模式 → 改写

## 四、发明内容

### 子节 1：发明目的

**结构**：
1. 目的句（1 段）："本发明的目的是提供一种 [方法名]，以解决上述现有技术存在的问题。"
2. 引入句（1 段）："本发明的优势体现在："
3. 优势条目（每条以分号结尾，最后一条句号）：
   - "（1）...；"
   - "（2）...；"
   - ...
   - "（N）...。"
4. 综述收尾句（可选）

### 子节 2：技术解决方案

**结构**：
1. 开场句：引用附图，给出大致流程
2. 主要步骤：每步独立成段，分号结尾，最后一步句号
3. "可选的"从属特征展开（3-5 组）

**所述使用规则**（**关键**）：
- 首次提及某技术要素：不加"所述"
- 后续引用：必须加"所述"
- 例：
  - 第一次："通过运动恢复结构算法获取初始稀疏点云"
  - 第二次："将所述初始稀疏点云输入..."

### 子节 3：技术效果

**结构**：
1. 引入句："本发明的技术效果为："
2. 效果描述：

格式 A（一段话，分号分隔）：
```
本发明通过 [手段A]，[效果A]；通过 [手段B]，[效果B]；通过 [手段C]，[效果C]。
```

格式 B（每条独立成段，"针对...通过..."模式）：
```
针对 [问题A]，本发明通过 [手段A]，[效果A]。

针对 [问题B]，本发明通过 [手段B]，[效果B]。
```

**Pass D 检测项**：
1. 出现公式（$...$ 或 $$...$$）→ 公式必须移到具体实施方式
2. "所述"使用错误（首次提及加了，或后续提及没加）→ 修正
3. 步骤未用分号串联 → 修正
4. 技术效果与发明目的字面镜像 → 改写技术效果展开机理
5. 技术效果列出空泛形容词 → 改写为"通过 X 实现 Y"格式

## 五、附图及附图的简单说明

**结构**：
1. 法律声明 1（"为了更清楚地说明..."）— 不动
2. 法律声明 2（"构成本申请的一部分的附图..."）— 不动
3. 逐图描述：

固定格式：
```
图X 为本发明实施例 [中的/提供的] [描述] [示意图/流程图]。
```

**禁用**：
- "所述" — 附图说明中绝对不使用所述
- 引号包裹技术名词
- 解释性文字（"该图展示了..." → 删除，只写图的名称）

**标点规则**：
- 中间图：分号结尾
- 最后一图：句号结尾

**Pass D 检测项**：
1. 图描述行使用了"所述" → 删除
2. 图描述行有多余解释 → 精简到固定格式
3. 标点不一致 → 统一

## 六、具体实施方式

**结构**：
1. 法律声明 1-5（5 段）— 不动
2. 实施例概述段
3. 详细步骤（主体，占 80% 篇幅）
4. 实施例技术效果段
5. 实施例总结段
6. 结尾固定段（"以上所述..."）— 不动

**所述使用规则**：
- 频率**大幅降低**，改用"上述"、"该"、直接指代
- 这是与发明内容章节最显著的差异
- AI 容易把发明内容的"所述"密度照搬到具体实施方式 → 改写

**步骤命名规则**：
- 用功能名词短语（"图像采集与预处理"），不用动宾短语（"采集并预处理图像"）
- 编号格式：（1）/（2）/（3）... 或 步骤S1/S2/S3...，**同一篇专利内统一**

**子步骤格式**：
- a）b）c）d）（小写字母 + 右半圆括号）
- 不用 (a)、A.、1.

**公式规则**：
- 独立公式行末标注 `（X）`（全角括号），从（1）开始连续递增
- 引用公式用"如式（X）"或"式（X）"
- 公式后紧跟"其中，..."的变量释义

**Pass D 检测项**：
1. "所述"密度过高（每 100 字 > 3 处）→ 改写为"上述/该/直接指代"
2. 步骤标题为动宾短语 → 改为名词短语
3. 子步骤编号不是 a）b）c）格式 → 修正
4. 公式编号不连续或缺失 → 修正
5. 公式引用未用"如式（X）" → 修正

## 七、权利要求书

**铁律**：本节在交底书中**完全不存在**。CNpatent skill 会主动删除该节。
若 humanizer 收到的文本包含"七、权利要求书"，应当：
1. 标记为错误：交底书不应包含权利要求书
2. 不处理该节内容
3. 在审查报告中提示用户

---

## 跨章节通用规则

无论在哪一节，以下规则普遍适用：

1. **不使用第一人称**：删除"我们"、"笔者"、"本文" → 改为"本发明"
2. **不使用"是的/确实/事实上"**：删除
3. **不使用感叹句、设问句、反问句**（背景技术结尾的"如何 X" 设问式除外）
4. **不使用括号补充说明**（"（注：...）"）→ 改写为正文
5. **不使用引号包裹技术术语**：除了首次定义新名词
6. **数字一律用阿拉伯数字**（除"一种"、"一个"等量词）
7. **单位与数字之间不加空格**（"30ms" 不写"30 ms"）
8. **百分号紧跟数字**（"30%" 不写"30 %"）

## 章节判断辅助

如果 humanizer 收到的是单段文本而非完整文档，需要根据特征判断所属章节：

| 特征 | 判断为 |
|---|---|
| 单句话，"本发明属于..." | 二、技术领域 |
| 段落讨论现有技术问题，无图引用 | 三、背景技术 |
| 段中含分号串联的步骤、含"所述"高频 | 四、发明内容·技术解决方案 |
| 段含"本发明的优势体现在" | 四、发明内容·发明目的 |
| 段含"本发明的技术效果为" | 四、发明内容·技术效果 |
| 单行格式 "图X 为本发明实施例..." | 五、附图说明 |
| 段落含 LaTeX 公式 | 六、具体实施方式 |
| 段落含具体参数值（如 K=20） | 六、具体实施方式 |
| 段落含子步骤 a）b）c） | 六、具体实施方式 |

如果无法判断，应当请用户提供章节上下文，不要默认按"通用文本"处理。
