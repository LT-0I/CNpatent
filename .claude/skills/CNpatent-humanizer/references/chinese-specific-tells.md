# 中文独有 AI 痕迹模式

> 本文档收录**中文文本特有**的 AI 痕迹模式，这些模式在英文 humanizer
> （如 blader/humanizer）中不存在或表现不同。
>
> 主要参考：[hong111109/humanizer](https://github.com/hong111109/humanizer)
> 的中文 28 类检测中适用于专利的部分（删除"语气词缺失"、"成语堆叠"
> 中的不适用条目），以及 GPTZero 的 burstiness 概念。

---

## 一、动词名词化（最高频中文 AI 痕迹）

中文 AI 倾向把动词改写成"进行 + 名词"的结构，因为这样听起来更"正式"。
但在专利中，**直接动词更准确**。

### 检测正则
```regex
进行(?:[\u4e00-\u9fa5]{1,4})|对.+进行(?:[\u4e00-\u9fa5]{1,4})|做出(?:[\u4e00-\u9fa5]{1,3})
```

### 重写示例

| AI 写法 | 专利写法 |
|---|---|
| 对图像进行预处理 | 预处理图像 / 对图像预处理 |
| 对点云进行补全 | 补全点云 |
| 对模型进行训练 | 训练模型 |
| 对结果进行分析 | 分析结果 |
| 进行特征提取 | 提取特征 |
| 进行误差计算 | 计算误差 |
| 进行参数优化 | 优化参数 |
| 对数据进行处理 | 处理数据 |
| 做出判断 | 判断 |
| 做出选择 | 选择 |

### 改写规则
- "对 X 进行 Y" → "对 X Y" 或 "Y X"
- "进行 X" → "X"（直接用动词）
- 例外：当 "进行" 后是双音节/三音节抽象名词，且改写会丢失语义时保留
  - 例："进行训练"中"训练"既是动词也是名词，改为"训练"语义完整
  - 例："进行对比实验"改为"对比实验"语义模糊，可保留"进行"

### 阈值
- Tier 2 词汇：单个不算问题，全文 ≥10 处才算
- Tier 3 词汇："进行" 字密度 > 1% 时也算

## 二、成语堆叠

AI 倾向连续使用四字成语来"显得有文化"，但专利**避免成语**，因为成语
含义模糊，不利于权利要求清晰性。

### 检测方法
1. 加载常见成语词典
2. 找连续 ≥2 个成语之间间隔 < 5 个字符的位置
3. 标记为成语堆叠

### AI 写法示例
```
本发明匠心独运、精益求精，集成了诸多技术于一体，可谓相得益彰。
```

### 专利写法
```
本发明集成了 X 技术、Y 技术和 Z 技术，提供一种 ABC 方法。
```

### 专利中应当**全部删除**的成语清单
- 匠心独运、精益求精、独具匠心、巧夺天工
- 相得益彰、相辅相成、画龙点睛
- 一目了然、显而易见、不言而喻、毋庸置疑
- 锦上添花、雪中送炭、独树一帜
- 包罗万象、面面俱到、应运而生
- 与时俱进、推陈出新

### 例外
- "现有技术"是否包含"现有"+"技术"两个成语？不，这是固定术语，**保留**
- "技术问题"、"技术效果"、"技术方案"等固定术语**保留**

## 三、中英标点混用

AI 在生成中文时，标点常常一会儿用中文一会儿用英文，特别是逗号、引号、
括号。专利文档应**全部使用中文标点**。

### 检测正则
```regex
[\u4e00-\u9fa5][,;:!?]\s|[a-zA-Z][，；：！？]
```

### 替换映射

| AI 写法（混用） | 专利写法 |
|---|---|
| 中文, 中文 | 中文，中文 |
| 中文; 中文 | 中文；中文 |
| 中文: 中文 | 中文：中文 |
| 中文(中文) | 中文（中文） |
| "中文" | "中文"（不使用 ASCII 双引号） |
| 'X' | "X"（不使用 ASCII 单引号） |

### 例外
- LaTeX 公式中的标点（`,`, `;`）保持英文
- 英文术语缩写中的标点（如 "U.S."）保持英文
- 数字和单位之间不加空格（"30ms"）

## 四、低突发度（Low Burstiness）

来自 GPTZero 的 burstiness 概念：人类写作的句长方差大，AI 写作的句长方差小。

### 定义
对一段文本，将句号、分号作为分句符，得到句子列表。计算每 4 个连续句子
的字数标准差除以均值（变异系数 CV）：

```python
def compute_burstiness(sentences, window=4):
    """
    返回每个滑动窗口的 CV 值列表。
    CV < 0.15 表示低突发度（疑似 AI）
    """
    cvs = []
    for i in range(len(sentences) - window + 1):
        window_lens = [len(s) for s in sentences[i:i+window]]
        if not window_lens:
            continue
        mean = sum(window_lens) / len(window_lens)
        if mean == 0:
            continue
        variance = sum((l - mean) ** 2 for l in window_lens) / len(window_lens)
        std = variance ** 0.5
        cv = std / mean
        cvs.append(cv)
    return cvs
```

### 阈值
- CV < 0.15：标记为"低突发度"，疑似 AI
- CV ≥ 0.15 且 < 0.30：边缘
- CV ≥ 0.30：人类节奏

### 修复方法
- 合并相邻短句为一句
- 将超长句在自然连接词处拆分
- 制造长短交错的节奏

### 专利场景的特殊性
专利的"具体实施方式"部分本身就有较多长复合句（描述步骤），因此突发度
本来就低于通用文本。不应直接套用通用阈值，建议：
- 通用文本：CV < 0.20 标记
- 专利"具体实施方式"：CV < 0.12 标记
- 专利"背景技术"：CV < 0.18 标记

## 五、整齐的"四字短语对仗"

AI 倾向写出工整的对仗结构：

### AI 写法
```
本方法结构清晰，操作简便，应用广泛，前景广阔。
```

### 检测
连续 ≥3 个四字短语，每个之间用逗号分隔。

### 专利写法
直接陈述事实：
```
本方法包括 5 个步骤，可在 RTX 3090 上以 30 FPS 运行。
```

## 六、过度使用括号注释

AI 倾向用长括号补充说明，导致句子被"打断"。专利中**应避免**长括号，
改为独立句或正文。

### 检测
括号内容长度 > 20 字符。

### AI 写法
```
本发明采用神经辐射场（一种基于隐式表达的三维重建方法，由 Mildenhall 等
在 2020 年提出，能够通过多视角图像学习场景的体密度和颜色函数）进行点云补全。
```

### 专利写法
```
本发明采用神经辐射场进行点云补全。所述神经辐射场为本领域公知技术，
基于隐式表达对场景的体密度和颜色函数进行学习。
```

### 例外
括号中的简短补充（如"（1）"编号、"（K=20）"参数）**保留**。

## 七、虚词堆积

AI 中文倾向堆积虚词："的"、"了"、"和"、"与"、"以"、"在"。

### 检测方法
- "的" 字密度 > 6% → 标记
- 一段中"和"+"与"+"以及" 总数 > 5 → 标记
- "在...中" 结构 > 5 处 → 标记

### 修复方法
- 删除可有可无的"的"
  - "本发明的提供的方法" → "本发明提供的方法"
  - "进行的操作" → "操作"
- 用顿号替代"和"/"与"
  - "X 和 Y 和 Z" → "X、Y 和 Z"

## 八、"X 化处理" 模式

AI 倾向把动词改成"X 化"+"处理"的结构。

### 检测正则
```regex
[\u4e00-\u9fa5]化处理|[\u4e00-\u9fa5]化操作
```

### 重写

| AI 写法 | 专利写法 |
|---|---|
| 标准化处理 | 标准化 |
| 归一化处理 | 归一化 |
| 量化处理 | 量化 |
| 优化处理 | 优化 |
| 简化处理 | 简化 |

---

## 九、副词堆叠

AI 倾向堆叠副词：

### AI 写法
```
非常显著地大幅提升了模型性能。
```

### 修复
全部删除副词，保留核心动词：
```
提高了模型性能。
```

如果有具体数据，加入：
```
PSNR 从 25.2 dB 提高至 28.5 dB。
```

## 十、"X 和 Y 之间" 模式

AI 倾向把两个事物的关系写成"X 和 Y 之间的 Z"，专利更直接。

### AI 写法
```
点云和图像之间的对应关系
```

### 专利写法
```
所述点云与所述图像的对应关系
```

或更直接：
```
点云-图像对应关系
```

## 检测与修复一览

| 模式 | 检测正则 / 方法 | 修复方法 | 等级 |
|---|---|---|---|
| 动词名词化 | `进行(?:.{1,4})` | "进行 X" → "X" | Tier 2 |
| 成语堆叠 | 连续 ≥2 成语 | 删除 | Tier 1 |
| 中英标点混用 | `[中文][,;:]` | 统一中文标点 | Tier 1 |
| 低突发度 | 句长 CV 计算 | 长短句交错 | 风格 |
| 四字对仗 | 连续 ≥3 四字短语 | 改为陈述句 | 风格 |
| 长括号注释 | `（.{20,}）` | 拆为独立句 | Tier 2 |
| 虚词堆积 | "的"密度 / "和与以"计数 | 删除冗余虚词 | Tier 3 |
| X 化处理 | `[\u4e00-\u9fa5]化处理` | 删除"处理" | Tier 1 |
| 副词堆叠 | 连续 ≥2 副词 | 删除副词 | Tier 1 |
| X 和 Y 之间 | `.+和.+之间` | 改为更直接的表达 | Tier 2 |

## 综合调用顺序

在 Phase 1D 中，按以下顺序检测：
1. 中英标点混用（最容易修复，先做）
2. 动词名词化
3. X 化处理
4. 副词堆叠
5. 四字对仗
6. 成语堆叠
7. 长括号注释
8. 虚词堆积（密度检测，放在最后）
9. 低突发度（计算密集，放在最后）
10. X 和 Y 之间模式
