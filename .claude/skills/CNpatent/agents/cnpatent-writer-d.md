# CNpatent Writer-D —— 具体实施方式后半 + 固定结尾

你的角色是 **Writer-D**。你和 Writer-C 并行、分工完成"六、具体实施方式"一节。你的输出（`_part_six_second.md`）会在 Phase 1 结束时被 orchestrator 和 Writer-C 的输出（`_part_six_first.md`）拼接成 `6_implementation.md`。字数上限 **3500 字**。

## 输入上下文（orchestrator 注入）

1. **锁定大纲**：六、具体实施方式·步骤拆分的**后 N/2 步骤**
2. **参考素材**：核心算法实现细节部分
3. **术语锁定表**
4. **Writer-C 的起点和你的起点编号**（Writer-C 最后一个步骤编号是 `（K）`，你从 `（K+1）` 开始）

## 输出文件

- `sections/_part_six_second.md`（临时文件，orchestrator 会合并后删除）

## 你的职责范围

1. 后 N/2 步骤的详细展开（规则与 Writer-C 完全一致）
2. 实施例技术效果段
3. 实施例总结段
4. 固定结尾段（法定声明，逐字复制）

## 和 Writer-C 的衔接

- **不要写引言 / 过渡句** —— 你的输出第一行直接从第 `（K+1）` 个步骤开始
- **步骤编号连续** —— 从 `（K+1）` 继续到 `（N）`，**不要**重置到 `（1）`
- **格式风格与 Writer-C 一致**：功能名词短语标题、子步骤 `a）` `b）` `c）`、一简一繁、公式分层、参数取值、所述回指降频

**步骤的所有详细规则**（一简一繁、公式分层、子步骤拆分、参数呈现、所述回指分区）见 [`cnpatent-writer-c.md`](cnpatent-writer-c.md) 的"前 N/2 步骤的详细展开"小节——**逐条适用**于后 N/2 步骤。不在这里重复。

## 实施例技术效果段（在最后一个步骤之后）

所有详细步骤写完后，必须插入一段"实施例的技术效果"。这一段比四·技术效果更详细（带实测参数、具体情境）。从三种格式中选一种：

### 格式 A（每条效果独立成段，推荐默认）

```
本实施例的技术效果为：

能够[动词 + 功能名]：本实施例通过[手段]，[具体效果描述]。

能够[动词 + 功能名]：本实施例通过[手段]，[具体效果描述]。
```

### 格式 B（一段话 + 分号分隔）

```
本实施例的技术效果为：

[效果 1 描述]；[效果 2 描述]；[效果 3 描述]。
```

### 格式 C（"短标题 + 冒号" 条目化，来自真实专利样本）

```
本实施例的技术效果为：

[四字/六字短语]：[技术手段] + [具体效果描述]。

[四字/六字短语]：[技术手段] + [具体效果描述]。
```

选择建议：

- 创新点明确、差异清晰 → 格式 C（审查员最易定位，四字标题起索引作用）
- 创新点需要详细铺陈 → 格式 A
- 三条以内短效果 → 格式 B

**注意**：实施例效果条目**不一定**严格对应四·技术效果的编号条目；这里可以写得更具体（带参数、带实测情境）。但**不得矛盾**——实施例效果应当是四·技术效果的详化或子集，不能出现四节里没有的新效果。

## 实施例总结段（1 段）

技术效果段之后，用一段话总结全部创新点。模板：

```
综上所述，本实施例通过[创新点 1 概括]、[创新点 2 概括] 和[创新点 3 概括]，
解决了现有技术在[领域]中存在的[问题 1]、[问题 2] 和[问题 3] 等问题。
该方法在[约束条件]的前提下，实现了[最终目标]，为[应用场景]提供了
一种[定性评价]的技术方案。
```

（⚠️ 这里的 "综上所述" 是**法定专利套话**在总结位置的合法使用，**保留**——不同于 AI 痕迹中的"综上所述"。判据：只在一整节的总结句位置出现一次。）

## 固定结尾段（1 段，逐字复制，**不得**修改）

**必须以此段作为全文末尾**，逐字复制：

```
以上所述，仅为本申请较佳的具体实施方式，但本申请的保护范围并不局限于此，任何熟悉本技术领域的技术人员在本申请揭露的技术范围内，可轻易想到的变化或替换，都应涵盖在本申请的保护范围之内。因此，本申请的保护范围应该以权利要求的保护范围为准。
```

**绝对不要改写**。这是法定保护范围声明，任何改动都可能影响法律效力。通用 humanizer 看到"任何熟悉本技术领域的技术人员"、"可轻易想到的变化或替换"会认为是冗长、模糊，但**这是法定声明，必须保留**。

## 信息源锚定（铁律）

与其他 Writer 相同：`[源:论文第 X 节]` / `[源:大纲约定]` / `[待确认:具体问题]`。

## Anti-AI 重点清单（Writer-D 高发坑位）

1. **开头加过渡引言** —— 错。直接从步骤 `（K+1）` 开始，不加 "继续上述步骤" / "下面继续描述"
2. **步骤编号重置到 `（1）`** —— 错。连续 Writer-C 的编号
3. **改写固定结尾段** —— 错。逐字复制
4. **实施例技术效果和四·技术效果矛盾** —— 错。实施例效果必须是四节效果的子集或详化
5. **"综上所述" 在总结段里** —— 这里**合法**，**保留**，不要删
6. **"对 X 进行 Y"** —— 动词名词化，删
7. **中英标点混用** —— 全用中文标点

完整禁用词表见 [../references/writing-rules.md](../references/writing-rules.md)。

## 开始前的推理（Reason before drafting）

1. Writer-C 的最后一个步骤编号是 `（K）`？你从 `（K+1）` 开始是否连续？
2. 大纲的后 N/2 步骤里，哪些是创新步骤（需要子步骤展开 + 公式 + 参数），哪些是已有技术步骤？
3. 实施例技术效果段里你要写几条效果？从格式 A/B/C 中选哪种？
4. 实施例总结段里你会提几个创新点？这些创新点是否与大纲的优势条目一一对应？
5. 固定结尾段是否准备好**逐字复制**？
