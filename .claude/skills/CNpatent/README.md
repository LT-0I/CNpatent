# CNpatent — 中国发明专利技术交底书生成器

一个 [Claude Code Skill](https://docs.anthropic.com/en/docs/claude-code)，将学术论文、研发笔记、开源项目等参考素材转化为符合 CNIPA 规范的专利技术交底书（.docx）。

## 功能特点

- **模板驱动生成** — 加载内置专利模板，保留页眉页脚、页边距、字体等格式
- **领域驱动微创新** — 要求用户指定目标应用领域，用于场景迁移与专利查重规避
- **多 Agent 并行架构** — Planner 生成大纲 → 4 个 Writer Agent 并行写作 → Reviewer Agent 自动审查，大纲确认后全程无需人工干预
- **三层去 AI 痕迹** — 写作预防 + 正则自动替换 + Reviewer 终审 + `humanizer` skill 润色
- **信息源锚定（防幻觉）** — 所有技术内容追溯到参考素材，无法确认的参数标记 `[待确认]`，杜绝内容编造
- **真实专利行文经验** — 基于 5 份真实中国专利文档提取的写作模式（漏斗型背景技术、问题-方案交叉叙事、功能名词短语步骤命名、分区"所述"回指策略等）
- **防幻觉附图提示词** — 静默生成空间锚定的 AI 生图提示词，附带严格文字排他警告
- **纯文本 LaTeX 公式** — 所有数学公式以纯文本写入，防止格式损坏

## 前置依赖

### Python 环境

```bash
pip install python-docx
```

### 依赖的 Skill

| Skill 名称 | 用途 | 必需 |
|------------|------|:----:|
| **humanizer** | 去除 AI 写作痕迹，使文本符合专利文书风格 | 是 |
| **pdf** | 读取用户提供的 PDF 格式参考素材（论文、技术文档等） | 是 |
| **docx** | 读取用户提供的 .docx 格式参考素材或已有专利草稿 | 是 |

## 必需输入

| 输入项 | 必需 | 说明 | 示例 |
|--------|:----:|------|------|
| **参考素材** | 是 | 学术论文、技术文档、开源项目等 | PDF / .docx / 文本 / 链接 |
| **目标应用领域** | 是 | 专利要落地的具体工程场景 | 肛瘘术后创口三维重建 / 工业焊缝检测 |
| 现有专利参考 | 可选 | 同领域已有专利，用于差异化设计 | 专利号 / 标题 / 摘要 |

## 安装方式

### 方式一：项目级安装（推荐）

将 `CNpatent/` 目录放到项目的 `.claude/skills/` 下：

```
your-project/
└── .claude/
    └── skills/
        └── CNpatent/
            ├── SKILL.md
            ├── README.md
            ├── LICENSE
            ├── assets/
            │   └── 专利交底书模板.docx
            └── references/
                ├── docx-patterns.md
                └── writing-rules.md
```

### 方式二：全局安装

将 `CNpatent/` 目录复制到用户级 Claude 配置目录：

```
~/.claude/skills/CNpatent/
```

## 使用方法

在 Claude Code 中提供参考素材和目标领域：

```
帮我写一份专利技术交底书，参考这篇论文，目标领域是工业焊缝检测
```

```
把这篇论文转成专利交底书，应用场景是自动驾驶障碍物识别
```

如未指定目标领域，Skill 会主动询问。

## 执行流程

```
Phase 0  输入确认 → 场景迁移 → 结构化大纲生成
           ↓
         用户确认大纲（唯一确认点）
           ↓
Phase 1  自动任务拆分 → 4 个 Writer Agent 并行生成
           ↓
Phase 2  Reviewer Agent 自动三重审查（一致性 + 防幻觉 + 去AI味）
           ↓
Phase 3  整合 → final_deai_cleanup() 兜底替换 → 写入 DOCX
           ↓
Phase 4  DOCX 写入后自动验证（附图编号连续性、引用一致性）
           ↓
Phase 5  静默生成 AI 附图提示词文档
           ↓
Phase 6  附图修正（按需触发）
```

## 输出文件

| 文件 | 说明 |
|------|------|
| `[专利名称]_专利技术交底书.docx` | 完整专利技术交底书 |
| `[专利名称]_全套AI生图提示词.docx` | 防幻觉 AI 附图生成提示词 |

## 文档结构（电学类交底书格式）

```
【封面页】
  信息表（24行×4列，发明人/申请人基本资料）
  交底书注意事项（8段，原样保留）

【正文六节】
  一、发明/实用新型名称（≤25字）
  二、所属技术领域（1-2句）
  三、现有技术（背景技术）（现有技术概况 + 编号局限 + 引出句）
  四、发明内容：
     发明目的（总起句 + 编号优势 + 综述）
     技术解决方案（开场句 + 编号步骤概要）
     3、技术效果（总起句 + 编号效果 + 综述）
  五、附图及附图的简单说明（逐图一行描述）
  六、具体实施方式（实施例开场 + 详细步骤 + 公式 + 技术效果 + 总结 + 结尾）

  —— 七、权利要求书（已删除）——
```

注：权利要求书由专利代理人根据本交底书另行撰写，交底书中不包含。本格式不含说明书摘要。

## 核心技术特性

### 多 Agent 架构

| Agent | 职责 | 字数上限 |
|-------|------|---------|
| Writer-A | 一、发明名称 + 二、技术领域 + 三、背景技术 | ~1200字 |
| Writer-B | 四、发明内容（发明目的 + 技术解决方案 + 技术效果） | ~3500字 |
| Writer-C | 五、附图说明 + 六、具体实施方式 · 前半 | ~3500字 |
| Writer-D | 六、具体实施方式 · 后半 + 总结 | ~3500字 |
| Reviewer | 一致性 + 防幻觉 + 去AI味 三重审查 | — |

所有 Agent 使用 opus 模型，确保输出质量一致。

### 三层去 AI 痕迹

| 层级 | 机制 | 说明 |
|------|------|------|
| 第一层 | 写作预防 | Writer prompt 注入禁用词表，写作时即避免 |
| 第二层 | 正则替换 | `final_deai_cleanup()` 硬编码替换，不依赖 AI 判断 |
| 第三层 | Reviewer 终审 | 禁用词残留扫描 + 句式结构检测 + `humanizer` skill |

### 真实专利行文经验

基于 5 份中国发明专利文档（3 份交底书 + 2 份公开专利）提取的写作模式：

- 背景技术"漏斗式"段落节奏（短→长→长→短问题句）
- 发明内容"问题-方案"交叉叙事
- "所述"回指分区策略（发明内容严格、具体实施方式放松）
- 步骤标题功能名词短语命名法
- 子算法引入句固定模板
- "一简一繁"描述策略（已有技术简写、创新步骤详写）
- 公式"夹叙夹议"穿插物理直觉解读
- "短标题+冒号"技术效果条目化格式
