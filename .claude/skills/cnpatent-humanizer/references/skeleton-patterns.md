# 骨架模板目录 (Skeleton Patterns Catalog)

> 列出在真实中文专利失败样本中出现的 8-12 个**骨架级** AI 痕迹模板。
> 与 `chinese-specific-tells.md` 的表层词级痕迹互补：骨架模板描述的是
> 句群层面的修辞脚手架形状，**即使逐条替换词表也无法消除**。
>
> 每条骨架给出：模板、真实出处样本、对应的检测器、重写方向。

---

## 骨架 S-01: 状语前置并列效果骨架

**模板**:
```
[状语前置/地点/条件] + [主语] + [否定副词/情态动词] + [效果动词]
+ [：机制说明] + [；量化数据 optional]
```

**示例 (来自 4c_effect.md 去 AI 前)**:
```
（1）长距离巡检下几何先验场边界偏差不再单向累积：...
（5）退化场景下系统行为可预测、下游可识别：...
```

**检测器**: `skeleton_sim.py`
(signature: has_locative_prefix + has_neg_or_modal + has_colon)

**重写指导**:
- 破坏 `状语前置 + 否定/情态并列` 模式
- 让至少 2 条改为短陈述句，去掉 `长...下 / 退化...下` 状语
- 把 `不再单向累积` 换成 `边界偏差保持稳定`，把否定改成肯定
- 不要求每条都带机制 + 数据；允许有些条目只做主张

---

## 骨架 S-02: 否定-情态-肯定三段链骨架

**模板**:
```
[否定副词 不/未/非] + [情态副词 仍/依然/仍然] + [肯定动词可/能/得以]
```

常见变体：
- `未...仍可...`
- `不...依然能够...`
- `非...仍旧得以...`

**示例 (4c_effect.md 样本)**:
```
（2）稀疏、延迟、含噪三重观测障碍下仍可完成 ...
（4）闭环系统不出现过修正振荡 ...
```

**检测器**: `skeleton_sim.py` (`has_neg_or_modal=True` 在首 12 token)

**重写指导**:
- 在 5 条兄弟中最多保留 1 条此骨架
- 其他条目改为单一极性句：纯肯定陈述 或 独立的条件从句
- 把 `未...仍可` 拆成 `系统输出保持有效；测试条件包含 ...`

---

## 骨架 S-03: 机制-数据-对比三元并列骨架

**模板**:
```
[主张] + [通过/基于/经 X 机制] + [量化数据 Y] + [相较 Z 对比]
```

每条都把 `mechanism + data + contrast` 三槽位全部填满。

**示例 (4c_effect.md 样本)**:
```
（2）... 消融测试中 ...下降约 5 个百分点。
（3）... ；消融测试中 ...。
```

**检测器**: `argumentation_slots.py` (subtype='exhaustive')

**重写指导**:
- 论证槽位按兄弟分散：其中 1-2 条纯主张，1 条带机制，1 条带数据，
  1 条带对比
- 允许 `消融测试` 只出现在 1 条，不要在 3+ 条重复
- 允许短条目 `（4）过修正风险在实验中未被观察到。` 独立成立

---

## 骨架 S-04: 动机-问题-机制三元骨架

**模板**:
```
[为解决 ... / 针对 ... 问题] + [主语] + [通过 / 基于 ... 机制]
+ [实现 ...]
```

常见于 §3 背景技术末段 + §4 发明目的段。

**示例** (虚构但频见):
```
为解决长隧道场景下稀疏点云的几何先验退化问题，本发明通过构造
XXX 机制，实现 YYY 稳定性提升。
```

**检测器**: `argumentation_slots.py` (slots 同时命中
`motivation + mechanism + data`)

**重写指导**:
- §3 末段只保留 `motivation`
- §4 发明目的拆成独立动机陈述 + 独立机制陈述 (两个句号断开)
- 不要在同一自然段塞 3 个槽

---

## 骨架 S-05: 解决-针对双框架骨架 (§3/§4 高发)

**模板**:
```
解决 ... 问题 / 针对 ... 难题
```

连续出现在兄弟条目 → 问题枚举骨架化。

**示例 (常见 §3)**:
```
(1) 针对几何先验失效问题，...
(2) 针对数据稀疏问题，...
(3) 针对延迟导致的漂移问题，...
```

**检测器**: `argumentation_slots.py` (slot=motivation 在 3+ 条兄弟)

**重写指导**:
- 在 3 条问题枚举里只保留 1 条使用 `针对 ...`
- 其他改写为 `X 场景下 ... 不稳定` / `现有 Y 方案在 ... 时失效`
- 避免三段 `针对 ... 针对 ... 针对` 的骨架

---

## 骨架 S-06: 并列推销动词骨架

**模板**:
```
[打破 / 提升 / 保证 / 保障 / 为 ... 提供] + 宾语
```

常见变体：`打破瓶颈 / 提升效率 / 保证稳定 / 保障安全 / 为后续 ...
提供支撑`。

**示例**:
```
（1）打破几何先验场退化瓶颈：...
（2）提升点云稀疏场景下的重建精度：...
（3）保证长尾场景下的系统鲁棒性：...
（4）为后续语义分割提供稳定的几何支撑。
```

**检测器**: `skeleton_sim.py` (main_verb_pos=v; 3+ 兄弟同位置同POS)

**重写指导**:
- 这组动词属于商业宣传语族；`audit.py` Tier1 的
  `颠覆性 / 划时代 / 开创性` 也属于同一骨架的词层实例
- 合并同义动词：`打破 + 提升` 只保留其一
- 把 `为 X 提供 Y` 改为 `X 使用 Y` (主动)

---

## 骨架 S-07: 构成 X 链路 / 形成 Y 环路骨架

**模板**:
```
[构成 / 形成 / 组成] + [X] + [链路 / 环路 / 回路 / 体系 / 闭环]
```

专利喜欢以系统构成句收尾，AI 化表现为每条末尾都带此骨架。

**示例**:
```
（1）... 构成输入-处理-输出闭环。
（2）... 形成几何先验更新链路。
（3）... 组成前端-后端协同回路。
```

**检测器**: 目前未实装独立检测器。手动检查：
`grep -E '构成.{2,15}链路|形成.{2,15}环路|组成.{2,15}体系'`

**重写指导**:
- 至多一条用 `构成 ... 链路`；其他直接用动宾陈述：
  `输出结果写入 XYZ` 或 `YYY 回传至 ZZZ`
- 不要每个兄弟都收尾于「构成体系化描述」

---

## 骨架 S-08: 固定对称对比骨架

**模板**:
```
相较于 [现有 / 传统 / 单向] [方案 / 流水线 / 架构]，
本发明 [表达 X]; 较 [基线] 降低 / 提升 Y%
```

每条都在对比基线 → 论证对比槽填满。

**示例**:
```
（1）相较现有单向流水线，漂移被闭环抑制...
（2）相较基线方案，在稀疏场景下下降 8%...
（3）较传统架构，系统延迟降低 20%...
```

**检测器**: `argumentation_slots.py` (slot=contrast 在 3+ 兄弟)

**重写指导**:
- 至多 1-2 条用 `相较 / 较`；其他改为独立主张 (无显式对比)
- `基线`、`单向流水线`、`现有方案` 等引用至多出现 1 次
- 对比只在需要的地方放，不在每条都做 `相对 baseline 的百分比`

---

## 骨架 S-09: 段首「其一/其二/其三」骨架 (背景技术 §3)

**模板**:
```
现有技术存在以下问题：
其一 ...
其二 ...
其三 ...
```

**检测器**: `audit.py` Critical 正则 `其一.*其二` (已实装)

**重写指导**:
- 直接展开为三段问题陈述，不加 `其一 / 其二 / 其三` 序号词
- 也可用 `（1）（2）（3）` 但内部句式必须破骨架 (参考 S-01)

---

## 骨架 S-10: 发明目的-技术效果镜像骨架 (§4 跨节)

**模板**:
```
§4.1 发明目的: 为解决 P1 / P2 / P3 三类问题
§4.3 技术效果: 实现 E1 / E2 / E3 三项效果，与 P1/P2/P3 一一对应
```

AI 最强指纹：**E_i 是 P_i 的字面镜像**，每条都精确 1:1 对应。

**示例**:
```
§4.1: ...(1) 解决几何先验退化问题; (2) 解决数据稀疏问题;
       (3) 解决延迟漂移问题
§4.3: ...(1) 几何先验不再退化; (2) 数据稀疏被克服;
       (3) 延迟漂移被抑制
```

**检测器**: 余弦相似度 (P1) — 目前未实装；`reader_pass` agent 会看见。
此骨架是 reader-view 最容易识别的跨节镜像。

**重写指导**:
- §4.3 效果条数 ≠ §4.1 目的条数 (如目的 3 条，效果 5 条或 2 条)
- §4.3 某一条合并 2 个目的 (覆盖多个) 或引入目的未列的附带效果
- 避免词对词镜像：`解决延迟漂移 → 延迟漂移被抑制` 改为
  `闭环反馈时序在测试集内未观察到累积偏差`

## 骨架 S-11: 具体实施方式 §6 天然平行 (**不计分**)

**模板**:
```
(a) 步骤：...
(b) 步骤：...
(c) 步骤：...
```

子步骤本身可以形式并列，这是中国专利 §6 的合法写法。

**检测器**: `skeleton_sim.py + argumentation_slots.py` 在
`--section 六` 下标 `non_scoring=True`，不贡献分。

**重写指导** (仅当论证槽同时并列才重写):
- 如果每个 (a)(b)(c) 都带机制 + 数据 + 对比三元 → 触发
- 如果只是 (a)(b)(c) 动作描述 → **保留**，不视为 AI 骨架

## 骨架 S-12: 段尾总结句骨架

**模板**:
```
综上所述 / 总而言之 / 总之 / 综合来看 ...
```

单一词层 → 已在 `chinese-specific-tells.md` 和 Tier1 覆盖。
纳入此目录以提示 reader_pass 不要重复标记。

**检测器**: `audit.py` Critical (已实装)

**重写指导**: 整条删除；中国专利正文不写总结段。

## 使用约定

- 任何 `骨架 S-xx` 都应由 Phase 1 检测器 + Phase 4.5 reader_pass 双
  通道识别。
- 章节差异化：§6 对 S-11 豁免；§1/§2 很少触发骨架 (内容短)；
  §3/§4 是骨架高发区。
- 与 `patent-anti-patterns.md` 的「信息密度均匀」章节互补：
  anti-patterns 讲问题 + 例子，本目录讲模板形状 + 检测器映射。
- 重写只破骨架，不改术语/数值/专利号；参见 `do-not-overcorrect.md`。
