# 专利专属 AI 痕迹模式与重写示例

> 本文档收录中文专利文本中**特有的** AI 写作痕迹模式。这些模式是通用 humanizer
> 工具（如 op7418/Humanizer-zh）无法识别或会错误处理的，因为它们只在专利
> 这种**形式严格、术语锁定、句式固定**的文体中才表现为问题。

---

## 一、推销式形容词在技术描述中

通用 humanizer 已经识别"卓越的/优异的"等推销词，但在专利中，**任何主观
形容词修饰技术名词**都属于 AI 痕迹。

### 模式 1.1：技术效果主观化

**AI 写法**：
```
本发明的高斯泼溅算法具有较好的渲染质量和较高的运算效率，
能够在保证模型精度的同时显著降低计算开销。
```

**问题**：
- "较好的"、"较高的"、"显著"都是主观判断
- "在保证 X 的同时降低 Y" 是平衡式 AI 套话

**专利写法**：
```
本发明的高斯泼溅算法在 RTX 3090 上对 512×512 分辨率图像的渲染速度为
30 帧/秒，相比基础三维高斯泼溅方法，单帧推理时间从 56ms 降至 33ms。
```

**改写规则**：用具体指标替代主观形容词。若指标不在参考素材中，则删除该
主观形容词，不补充虚构数据。

---

### 模式 1.2：套话结尾段

**AI 写法**：
```
综上所述，本发明的方法具有结构清晰、操作简便、效果显著、应用前景广阔
等优点，能够为相关领域提供有效的技术支撑。
```

**问题**：
- "综上所述" 是禁用词
- "结构清晰/操作简便/效果显著" 是空泛形容
- "应用前景广阔" 是禁用词
- "提供有效的技术支撑" 是套话

**专利写法**：
```
本实施例通过激光雷达轴向退化检测和图像光照质量评估双重判据将传感器融合
策略与 ARM 嵌入式平台的实时运行状态关联，解决了现有技术在资源受限平台
上隧道场景定位面临的算力瓶颈、几何退化和内存不可控增长问题。
```

**改写规则**：删除"综上所述"，将"具有 X、Y、Z 等优点"改为"通过 [具体技术
手段]，解决了 [具体问题]"——优点用动词陈述，不用形容词列表。

---

## 二、AI 倾向的"完美对称"结构

通用 humanizer 的"三段式凑数"检测能识别恰好 3 条的列表。专利中要更进一
步：**条目数对称**也是 AI 痕迹。

### 模式 2.1：发明目的-技术效果完全镜像

**AI 写法**：
```
[发明目的]
本发明的优势体现在：
（1）解决了点云空洞问题；
（2）提高了渲染质量；
（3）降低了计算开销。

[技术效果]
本发明的技术效果为：
（1）解决了点云空洞问题；
（2）提高了渲染质量；
（3）降低了计算开销。
```

**问题**：发明目的和技术效果完全字面镜像，AI 习惯把两节写成同义复述。

**专利写法**：发明目的写"想做什么"，技术效果写"为什么这样做能达到这个
结果"。两节内容应该有**因果关系**而非**字面镜像**。

```
[发明目的]
本发明的优势体现在：
（1）通过基于立方体的神经辐射场对窦道深层盲区进行几何推断，解决了运动
恢复结构算法在凹陷区域产生的点云空洞问题；
（2）通过 PCA 法向量约束使高斯基元在初始化阶段即贴合粘膜壁面，提高了
弯曲表面的渲染质量；
（3）通过混合损失函数和自适应密度控制，降低了凹陷细节区域所需的迭代次数。

[技术效果]
本发明的技术效果为：

深层窦道几何推断：基于立方体的神经辐射场网络利用隐式空间表达能力，对运动
恢复结构算法无法覆盖的深层区域进行几何推断，为后续三维高斯泼溅提供了几何
完整性较高的初始化数据。

柔性曲面拟合：PCA 法向量约束使高斯基元在初始化阶段即呈贴合粘膜壁面的扁平
椭球形态，减少了从无约束球形向贴合曲面收敛所需的迭代次数。

边界保护与凹陷加密：D-SSIM 损失保护了粘膜半透明边界的结构特征，自适应
密度控制在凹陷区域增加高斯基元密度。
```

**改写规则**：技术效果应展开"机理"，而非重复发明目的的字面表述。

---

### 模式 2.2：步骤数恰为偶数或常见数字

**AI 痕迹信号**：技术解决方案恰好 4 步、6 步、或 8 步。真实专利的步骤数
往往是奇数（3、5、7、9），因为工程流程很少完美对齐。

**检测方法**：若步骤数为 4/6/8/10 且每步字数相近，告警。

**改写不强制改变步骤数**，但要检查：
- 是否有"凑数"步骤（如最后一步是"输出结果"或"完成可视化"）
- 是否将技术上紧密关联的两步硬性分开
- 是否将松散的几个操作硬性合并为一步

## 三、AI 倾向的填充式连接

### 模式 3.1：每段都用关联词开头

**AI 写法**：
```
首先，对图像进行预处理...
其次，使用运动恢复结构算法...
然后，构建空间体素剖分...
此外，为了提高精度...
最后，输出三维模型。
```

**问题**：
- "首先/其次/然后/此外/最后" 是禁用词
- AI 倾向给每段都加显式关联词

**专利写法**（去掉关联词，让逻辑隐含）：
```
对预处理后的图像序列利用运动恢复结构算法进行特征匹配与三角化，获得稀疏
点云及各帧图像对应的相机位姿参数；根据所述稀疏点云的空间分布范围构建
三维包围盒，对所述包围盒进行基于立方体的空间体素剖分采样；将每个体素
单元的中心坐标作为查询点输入神经辐射场网络，预测查询点的体密度与颜色
值；保留体密度超过阈值的查询点作为补全点云。
```

**改写规则**：在"技术解决方案"和"具体实施方式"中，**用分号串联步骤，
而不是用关联词分段**。

---

### 模式 3.2：因果链显式过密

**AI 写法**：
```
由于点云存在空洞，因此需要补全。为了补全空洞，采用神经辐射场。
由于神经辐射场需要相机位姿，因此先用 SfM 算法获取位姿。
```

**问题**：每一步都用"由于/因此/为了"显式连接，过于"教科书化"。

**专利写法**：
```
利用运动恢复结构算法对所述预处理后的图像序列进行特征匹配与三角化，获得
稀疏点云及各帧图像对应的相机位姿参数。根据所述初始稀疏点云和相机位姿
参数构建创口区域的三维空间包围盒，对所述包围盒进行基于立方体的空间体素
剖分采样。
```

**改写规则**：技术步骤之间的因果由**操作顺序**和**输入输出关系**自然表达，
不需要每步都加"由于/因此"。

---

## 四、AI 倾向的"全文均匀"密度

### 模式 4.1：信息密度均匀化

人写专利的特点：
- **创新步骤段**：信息密度极高，每句话都有具体技术内容
- **过渡段/法律声明段/章节引导段**：信息密度低
- 两类段落字数比通常 > 2:1

AI 倾向：每段信息密度接近平均值，没有明显的"高密度核心段"。

**检测方法**：
1. 统计每段的"技术名词密度"（每 100 字中专业名词数量）
2. 若全文段落的技术名词密度方差 < 全文均值的 0.2，告警

**改写方法**：
- 创新算法步骤段加入更多具体参数、子步骤、变量解释
- 过渡段精简为 1-2 句

---

### 模式 4.2：句子长度均匀化

**AI 痕迹**：连续 4 个以上句子，字数差异 < 15%。

**真实专利节奏**：长短句交错，例如：
- 短（10-15 字）：陈述操作
- 长（30-50 字）：展开方法、引入参数
- 短：给出参数值
- 长：解释物理意义

**检测方法**：滑动窗口，对每 4 句计算字数标准差/均值。

**改写方法**：合并相邻短句、拆分超长句。

---

## 五、AI 倾向的"不必要的同义改写"

### 模式 5.1：术语漂移

这是专利 humanizer **最重要的检测项**。详见 SKILL.md 的 Pass C。

**典型 AI 痕迹**（来自真实生成案例）：
| 第一次出现 | 第二次出现 | 第三次出现 |
|---|---|---|
| 稀疏点云 | 初始点云 | 原始点云 |
| 体素剖分 | 空间剖分 | 立方体划分 |
| 神经辐射场网络 | NeRF 网络 | 辐射场模型 |
| 高斯基元 | 高斯单元 | 高斯分布 |
| 损失函数 | 优化目标 | 训练指标 |

**改写规则**：全部统一为首次出现的术语。

### 模式 5.2：句首主语漂移

AI 倾向在连续段落中变换主语来"避免重复"：
- 第 1 段："本发明..."
- 第 2 段："该方法..."
- 第 3 段："所述方案..."
- 第 4 段："此技术..."

专利写法：在同一节内主语应保持稳定。"本发明" → "本发明" → "本发明"，
或者首次"本发明"之后用"所述方法"，后续保持"所述方法"。

## 六、AI 倾向的"温和升级"

### 模式 6.1：用比较级而非具体值

**AI 写法**：
```
本方法的精度比传统方法更高，鲁棒性更强，运行速度更快。
```

**问题**：
- 比较级是主观判断
- 没有"比哪个传统方法"的明确对照

**专利写法**：
```
本方法在 ScanNet 数据集上的 PSNR 为 28.5 dB，相比基础 3DGS 的 25.2 dB
提高 3.3 dB；在 RTX 3090 上单帧推理时间从 56 ms 降至 33 ms。
```

**改写规则**：删除比较级，若有具体数据（来自参考素材或大纲）则填入；
若无，则改为对操作的描述而非效果的判断。

---

## 七、AI 倾向的"为了平衡而平衡"

### 模式 7.1：每个创新点都配一个副作用提及

**AI 写法**：
```
本方法虽然计算开销略有增加，但精度显著提升。
```

**问题**：
- "虽然...但..." 是禁用句式
- "显著" 是禁用词
- "略有/略微" 是模糊词

**专利写法**：
```
本方法的单帧推理时间为 33 ms，PSNR 为 28.5 dB。
```

**改写规则**：去掉"权衡式"评论，专利只陈述事实，不做利弊评价。利弊评价
属于审查意见答复阶段，不属于交底书。

---

## 八、AI 倾向的"过度形容"

### 模式 8.1：动词被形容词替代

**AI 写法**：
```
本发明能够实现对图像的精细化处理、对特征的高效提取、对结果的准确预测。
```

**问题**：
- "实现对 X 的 Y 化处理" 是 AI 句型
- "精细化/高效/准确" 是空形容词
- 三段式凑数

**专利写法**：
```
本发明对图像执行预处理、特征提取与结果预测。
```

**改写规则**：用动词直接陈述操作，删除"实现对 X 的"和"对 X 进行 Y 化"
这类被动结构。

---

## 九、AI 倾向的"伪精确"

### 模式 9.1：含糊的具体数字

**AI 写法**：
```
本方法处理速度约为传统方法的 2 倍以上，精度提升约 30%。
```

**问题**：
- "约/大约" 是禁用词
- "X 倍以上" 是模糊比较
- "约 30%" 是无源参考的数字

**改写规则**：
- 若数据有出处：删除"约/以上"，写确定值
- 若数据无出处：标记 `[待确认:具体提升幅度]`，让 Reviewer 处理

---

## 十、AI 倾向的"陈述-总结"双重叙述

### 模式 10.1：每段末尾重述本段要点

**AI 写法**：
```
对图像进行预处理，包括去畸变、降噪和直方图均衡化。这些预处理操作有效
提升了图像质量，为后续特征提取奠定了基础。
```

**问题**：
- "这些 X 操作 Y 提升了 Z" 是 AI 总结句套路
- "为后续 X 奠定了基础" 是禁用句式

**专利写法**：
```
对图像进行预处理，包括去畸变、降噪和直方图均衡化。
```

**改写规则**：删除每段末尾的"总结句"，让事实自己说话。专利不需要"承上启下"
的过渡总结。

## 模式速查表

| 模式 ID | 类型 | 检测方法 | 修复方法 |
|---|---|---|---|
| 1.1 | 推销形容词 | 扫描"较好/较高/显著" | 用具体指标替代或删除 |
| 1.2 | 套话结尾段 | 扫描"综上所述" | 改为动词式效果陈述 |
| 2.1 | 对称镜像 | 比较发明目的与技术效果字面 | 技术效果展开机理 |
| 2.2 | 偶数步骤 | 步骤数为 4/6/8 | 检查是否凑数 |
| 3.1 | 关联词段首 | 段首扫描"首先/其次..." | 用分号串联 |
| 3.2 | 因果链显式 | 计数"由于/因此"密度 | 删减冗余因果词 |
| 4.1 | 密度均匀 | 段落技术名词密度方差 | 创新段加密、过渡段精简 |
| 4.2 | 句长均匀 | 4 句滑窗字数标准差 | 长短句交错 |
| 5.1 | 术语漂移 | 术语锁定表比对 | 统一为首次术语 |
| 5.2 | 主语漂移 | 段首主语聚类 | 保持主语稳定 |
| 6.1 | 比较级 | 扫描"更/比...更" | 用具体数据 |
| 7.1 | 权衡式 | 扫描"虽然...但..." | 删除评价 |
| 8.1 | 动词形容化 | 扫描"X 化处理" | 用直接动词 |
| 9.1 | 含糊数字 | 扫描"约/大约/以上" | 删除模糊词或标记待确认 |
| 10.1 | 段尾总结 | 扫描"为后续 X 奠定基础" | 删除总结句 |
