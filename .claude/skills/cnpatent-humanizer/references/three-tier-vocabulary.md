# 中文专利 3 级词汇模型

> 本文档定义专利场景下的三级词汇检测系统，借鉴 [avoid-ai-writing](https://github.com/conorbronsdon/avoid-ai-writing)
> 的 Tier 1/2/3 模型并完全中文化、专利化。
>
> **核心思想**：不是所有 AI 痕迹词都应该一刀切。有些词单次出现就是问题
> （Tier 1），有些词只在聚集时是问题（Tier 2），有些常见词只在密度过高时
> 才是问题（Tier 3）。这样可以**大幅减少误报**——单个"基于"不是 AI 痕迹，
> 但全文 50 处"基于"就可能是。

---

### 1.6 发明八股句（10 句，v1.4 新增）

专利 AI 生成文本的元话语特征句。这些句式在真实代理人撰写的交底书中近乎不出现，但 AI 几乎必带。单次出现即为 critical。

| 句式（正则友好模板） | 处置 |
|---|---|
| 本发明的目的是提供 | (删除，改为直接陈述方法) |
| 本发明的优势体现 | (删除整句) |
| 本发明提供.{0,20}相比现有技术具有 | (删除整句) |
| 本方法.{0,10}相比现有技术 | (删除对比句) |
| 本步骤.{0,10}核心技术创新 | (删除整句) |
| 本方案.{0,10}必要性 | (删除段落) |
| 本方案.{0,10}收敛性 | (删除段落) |
| 本方案.{0,10}效果.{0,10}佐证 | (删除段落) |
| 本发明的技术效果为 | (删除) |
| 综上所述.{0,20}本发明 | (删除) |

**Why**：这些是 AI 向读者"推销"发明而非陈述技术的标志。真实专利采用"A，B"式动作叙述，不说"本发明提供 A"。禁用后文本会自然转为技术第一、元话语零。

### 1.7 强调副词（9 词，v1.4 新增）

报告§15 证据：高频"强调性副词"是专利 AI 味的稳定指纹。任何一处修饰技术名词即删除。

| 词 | 规则 |
|---|---|
| 持续 | 技术名词前 → 删除（"持续追踪" → "追踪"；"持续累积" → "累积"）|
| 严重 | (删除) |
| 显著 | (删除；已在 1.1 覆盖复合词) |
| 根本 | (删除) |
| 固有 | (删除；"反馈系统固有的" → "反馈系统的") |
| 专门 | (删除) |
| 有机 | (删除；"有机组成部分" → "组成部分") |
| 整体 | 技术名词前 → 删除（"整体重写" → "重写"；"整体虚警率" → "虚警率") |
| 全局 | 技术名词前 → 删除（"全局残余偏差" → "残余偏差") |

**例外保留**：用作具体技术术语的场合（"全局坐标系"、"整体流程图"、"持续时间"）不删。规则以"紧跟技术名词 + 语义为修饰强化"为判定边界。

### 1.9 导语禁用（v1.5 新增，对应 v2 报告规则 5/10）

以下**导语短语**（无论段首还是句中）一律 Tier 1 替换/删除：

| 模式 | 规则 |
|---|---|
| 为解决上述.{0,20}问题，| 删除 |
| 为解决.{0,20}该问题，| 删除 |
| 为解决.{0,30}问题，| 删除 |
| ^为此[，,] | 删除（段首） |
| ^针对.{0,20}[，,].{0,5}本步骤 | 删除前缀 |
| 本步骤(利用\|通过\|将\|引入\|采用\|根据) | 删除「本步骤」 |
| 本实施例(定义\|采用\|利用) | 改为「采用」 |
| 本步骤为本发明的核心技术创新所在 | 删除整句 |

**Why**：这些短语是 AI "先声夺人"式写法，在真实专利中罕见。专利要么直接陈述操作（无需导语），要么把动机段和操作段分离（两段式）。

### 1.8 风格标签动词（14 词，v1.4 新增）

报告§9 列"AI 高识别度风格词"。这些词本身并非错误，但密集出现（段内 ≥2 / 全文 ≥5）即为 AI 特征。v1.4 中按 Tier 1（单次即替换）处理，因为它们几乎总是可以用平实动词替换。

| 词 | 替换 |
|---|---|
| 锚定 | 属于 / 来自 / 依据 |
| 锚定在 | 依据 / 来自 |
| 降维 | 简化为 / 压缩为 |
| 降维为 | 简化为 |
| 注入 | 施加 / 加入（动词时保留，名词化时删） |
| 阻尼注入 | 阻尼施加（名词化场景） |
| 兜底 | 回退 / 保底 |
| 退化兜底 | 退化处理 |
| 自举闭环 | 闭环 / 相互校正 |
| 同根验证源 | 同源验证 / 共同来源验证 |
| 同根 | 同源 / 同出 |
| 迁移 | 应用 / 用于 |
| 落于同一参数集合 | 属于同一参数集合 / 同属一个参数集合 |
| 共同形成 | 形成 |
| 由此打破 | 打破 |
| 由此构成 | 构成 / 形成 |
| 施加分布式反向修正 | 做分布式反向修正 / 进行反向修正 |
| 予以限制 | 限制 |

**Why**：这些词在真实代理人撰写的专利里极少出现（基于 15 篇授权专利抽样）。AI 之所以爱用，是把"动作 + 动作对象"组合名词化以显得专业。替换为平实动词后文本的技术含量不变，但 AI 指纹显著下降。

## Tier 2：聚集标记词（Cluster-Flagged）

单次出现**不**标记；同一段落内 ≥ 2 次或 500 字范围内 ≥ 3 次才标记。
约 40 个条目。

### 2.1 中性技术连接词

### 2.7 话题切换标志词（v1.5 新增，对应 v2 报告规则 2）

单段内出现 ≥ 2 个即 flag（`topic_switch.py` 检测）：

| 模式 | 处置 |
|---|---|
| `架构上，` / `观测上，` / `稳定性上，` / `操作上，` / `效果上，` / `性能上，` / `机制上，` / `原理上，` / `流程上，` | 段内同类 ≥2 次 → 强制断段 |

| 模式 | 处置 |
|---|---|

## Tier 3：密度标记词（Density-Flagged）

非常常见的词，不能简单删除。仅在全文密度超过阈值（默认 3%）时才标记。
约 15 个条目。

| 词 | 密度阈值 | 备注 |
|---|---|---|
| 的 | 6% | 中文最常用助词，正常密度 4-5% |
| 是 | 2% | "X 是 Y" 句式过多暗示 AI |
| 在 | 4% | 介词密度过高表示句式单一 |
| 和 | 2% | 并列连词，过多暗示列表过密 |
| 与 | 2% | 同上 |
| 等 | 1% | 列举省略词，过多表示懒惰 |
| 进行 | 1% | 动词名词化的标志 |
| 实现 | 0.8% | 抽象动词，过多暗示空洞 |
| 通过 | 1.5% | 过多暗示因果链显式过密 |
| 利用 | 1% | 过多暗示推销 |
| 系统 | 1% | 过多暗示泛化 |
| 方法 | 1.5% | 过多暗示空洞 |
| 该 | 1% | 过多暗示主语漂移 |
| 所述 | 2.5% (发明内容章节) | 过多即使在发明内容也是问题 |

**特别注意**："所述"在"四、发明内容"中合理使用频率高（2-3%），在
"六、具体实施方式"中应该 < 1%。本表的密度阈值需按章节调整。

## 检测代码示例

```python
import re
from collections import Counter

# Tier 1 词表（精简示例）
TIER1_WORDS = {
    '显著提升': '提高',
    '卓越的': '较高的',
    '颠覆性的': '',
    '值得注意的是': '',
    '综上所述': '',
    '首先': '',
    '其次': '',
    '本文': '本发明',
    # ... 完整列表见上
}

# Tier 2 词表
TIER2_WORDS = {
    '基于': {'scope': 'paragraph', 'threshold': 3},
    '通过': {'scope': 'paragraph', 'threshold': 3},
    '在...的过程中': {'scope': 'document', 'threshold': 3},
    '同时': {'scope': 'document', 'threshold': 5},
}

# Tier 3 词表
TIER3_WORDS = {
    '是': 0.02,
    '进行': 0.01,
    '实现': 0.008,
    '所述': 0.025,  # 章节相关
}


def detect_tier1(text):
    """Tier 1 检测：单次出现即标记"""
    hits = []
    for word, replacement in TIER1_WORDS.items():
        for match in re.finditer(re.escape(word), text):
            hits.append({
                'tier': 1,
                'word': word,
                'replacement': replacement,
                'position': match.start(),
            })
    return hits


def detect_tier2(text, paragraphs):
    """Tier 2 检测：聚集才标记"""
    hits = []
    for word, rule in TIER2_WORDS.items():
        if rule['scope'] == 'paragraph':
            for i, para in enumerate(paragraphs):
                count = len(re.findall(re.escape(word), para))
                if count >= rule['threshold']:
                    hits.append({
                        'tier': 2,
                        'word': word,
                        'count': count,
                        'paragraph': i,
                    })
        elif rule['scope'] == 'document':
            count = len(re.findall(re.escape(word), text))
            if count >= rule['threshold']:
                hits.append({
                    'tier': 2,
                    'word': word,
                    'count': count,
                    'scope': 'document',
                })
    return hits


def detect_tier3(text):
    """Tier 3 检测：密度标记"""
    hits = []
    total_chars = len(text)
    if total_chars == 0:
        return hits
    for word, threshold in TIER3_WORDS.items():
        count = len(re.findall(re.escape(word), text))
        density = (count * len(word)) / total_chars
        if density > threshold:
            hits.append({
                'tier': 3,
                'word': word,
                'count': count,
                'density': density,
                'threshold': threshold,
            })
    return hits
```

## 调优建议

如果在某类专利中误报率高，可以：
1. 把误报词从 Tier 1 降级到 Tier 2/3
2. 调整 Tier 2 的聚集阈值
3. 调整 Tier 3 的密度阈值

调优应该基于真实的专利样本统计，而不是凭直觉。建议 grep 真实代理人撰写
的专利样本，统计每个词的密度，作为基线。
