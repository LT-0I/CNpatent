# 中国专利法新颖性与创造性 —— cnpatent-noveltycheck

> 基于 **《专利法》** 和 **《专利审查指南 2023》**。Judge agent 的三步法推理依据本文件。

## 1. 新颖性（专利法第 22 条第 2 款）

### 法条原文

> 新颖性，是指该发明或者实用新型不属于现有技术；也没有任何单位或者个人就同样的发明或者实用新型在申请日以前向国务院专利行政部门提出过申请，并记载在申请日以后（含申请日）公布的专利申请文件或者公告的专利文件中。

### 三要素解析

1. **不属于现有技术** —— 主要判据
2. **无抵触申请** —— 次要判据（详见 §3）
3. 现有技术的定义：申请日以前在国内外为公众所知的技术

### 破坏新颖性的条件

> **单一对比文件完整披露全部技术特征**

这是严格的条件。允许的判定：

| 情况 | 结论 |
|---|---|
| 单篇完整披露全部特征 | **破坏新颖性** |
| 一个区别特征为"否"（未公开） | **具备新颖性** |
| 存在"部分"（等同手段） | 需分析，"等同"不破坏新颖性 |
| 多篇组合披露全部特征 | **具备新颖性**（不允许组合） |

### 判断原则

- **单篇对比**：只能用一篇对比文件，不允许组合
- **直接毫无疑义确定**：公开必须是明确的，不能靠推断
- **等同与完全相同的区别**：
  - 完全相同（如数值微小差异）→ 破坏新颖性
  - 等同（不同手段达成相同目的）→ 不破坏新颖性，可能破坏创造性

### 示例

**场景 A（破坏新颖性）**：

```
本方案: 基于 CNN 的图像分类方法, 使用 ReLU 激活函数
对比文件 CN123: 基于卷积神经网络的图像分类方法, 使用 ReLU 激活函数

结论: 破坏新颖性 (特征完全对应)
```

**场景 B（具备新颖性）**：

```
本方案: 基于 CNN 的图像分类, 使用 ReLU 激活
对比文件 CN123: 基于 CNN 的图像分类, 使用 Sigmoid 激活

结论: 具备新颖性 (激活函数不同)
但: 需在创造性环节分析 "ReLU 替换 Sigmoid" 是否显而易见
```

## 2. 创造性（专利法第 22 条第 3 款）

### 法条原文

> 创造性，是指与现有技术相比，该发明具有突出的实质性特点和显著的进步。

### 三步法（2023 审查指南）

#### Step 1 —— 确定最接近的现有技术

从所有对比文件中选出**最接近的**一篇。2023 指南的选择标准：

1. **与本发明的技术领域最相关**
2. **解决的技术问题最关联**（⭐ 2023 指南强调这一条作为首要条件）
3. **公开的技术特征最多**

**注意**：2023 修改指南强调"技术问题的关联性"，即使公开特征多的文件，如果技术问题不相关，也不应选为最接近现有技术。这是对 2010 版的重要修改。

#### Step 2 —— 确定区别特征与实际技术问题

1. **列区别技术特征**：本方案 vs 最接近现有技术，所有本方案独有的特征
2. **确定实际技术问题**：基于区别特征在本方案中的实际技术效果**重新确定**

**⭐ 2023 指南的核心修改**：

> 不是照搬说明书声称的技术问题，而是**基于区别特征能达到的技术效果，重新确定**

这个修改意味着：

- 审查员在审查时会自己重新算一遍技术问题
- 申请人不能靠"说明书技术问题写得大"来逃避
- 如果区别特征的实际效果只对应小问题，技术问题就要缩小

#### Step 3 —— 判断显而易见性

对区别特征，回答：

> 现有技术中是否存在"技术启示"，使本领域技术人员有动机将该区别特征应用于最接近现有技术，以解决重新确定的技术问题？

**技术启示的来源**：

1. **同一份对比文件的其他部分**（Step 1 选的那篇的其他段落）
2. **另一份对比文件**（其他命中文件里的披露）
3. **公知常识**（2023 新增证据类型：**教科书、技术词典、技术手册**）
4. **惯用手段**（本领域常见实现方式）

**结论**：

- 存在技术启示 → **显而易见 → 无创造性**
- 无技术启示 → **非显而易见 → 有创造性**

### 2023 指南的其他关键修改

| 条款 | 修改内容 | 影响 |
|---|---|---|
| 第二部分第四章 | 公知常识证据类型扩展 | 技术词典、手册纳入 |
| 第二部分第四章 | 最接近现有技术选择强调"技术问题关联性" | Step 1 判据优先级调整 |
| 第二部分第四章 | 实际技术问题的重新确定 | Step 2 更严格 |
| 第二部分第四章 | 紧急状态公开豁免后第三方再公开不重新计算 | 宽限期新规 |

## 3. 抵触申请

### 定义（专利法 22.2 后半句）

> 任何单位或者个人就同样的发明或者实用新型在申请日以前向国务院专利行政部门提出过申请，并记载在申请日以后（含申请日）公布的专利申请文件或者公告的专利文件中。

### 要素

- **他人** 在先申请（不是申请人自己的在先申请，那不算抵触申请，按禁止重复授权处理）
- **申请日** 早于本方案申请日
- **公开日** 晚于（或等于）本方案申请日
- **同样的发明**：技术方案实质相同

### 效果

- **破坏新颖性** ✓
- **不破坏创造性** ✗（这是与普通现有技术的关键区别）

### 检测难点

抵触申请在本方案申请时可能还**没有公开**（它在先申请，但在后公开）。因此：

- **免费库** 无法查到未公开申请
- **商业库** 如 incoPat 有"早期申请"或"未公开申请"字段，可以查
- **时间窗口** 近 18 个月（通常申请后 18 个月内公开）

### 本 skill 的处理

Phase B 的 Guide agent 会为用户生成**专门的抵触申请检索卡片**，使用 incoPat 的申请日字段限定近 18 个月 + 公开状态筛选。这是 incoPat 相对 Google Patents 等免费库的核心独特价值。

## 4. 新颖性宽限期（专利法第 24 条）

### 法条

> 申请专利的发明创造在申请日以前六个月内，有下列情形之一的，不丧失新颖性：
> （1）在国家出现紧急状态或者非常情况时，为公共利益目的首次公开的；
> （2）在中国政府主办或者承认的国际展览会上首次展出的；
> （3）在规定的学术会议或者技术会议上首次发表的；
> （4）他人未经申请人同意而泄露其内容的。

### 要点

- **时间** 6 个月，不是 12 个月（美国）
- **范围** 有限：紧急状态 / 政府展览会 / 学术会议 / 泄露四种
- **重点** ⚠️ **申请人自己在论文、博客、产品发布会上的公开 不算**

这与美国、日本、欧洲的规则**不同**：

- **美国** 自己在任何地方发表论文后 12 个月内申请仍享有宽限期
- **日本** 自己学术发表 6 个月
- **中国** 自己发表不享宽限期（除非通过 24 条规定的学术会议）

### 程序要求

- **申请时** 主动声明要求宽限期
- **2 个月内** 提交证明文件

### 第 3 项"学术会议"的严格定义

> 在 **国务院有关主管部门** 或 **全国性学术团体组织** 召开的学术会议或技术会议上首次发表

不包括：

- 国外学术会议（ICML / NeurIPS / CVPR 等国际会议不算）
- 网络会议 / Zoom 研讨会
- 商业会议 / 厂商论坛
- 地方学会的小型会议（需核实是否属于全国性学术团体）

### 本 skill 的处理

Screener 读入时应询问用户：

- 方案是否已公开过？
- 公开日期、形式、场合？

如果用户在 6 个月外有自己的公开，Screener 应**红灯警告**：方案的新颖性已失效，无法通过中国专利申请（除非是 24 条规定的情形）。

### 2023 指南的新规

紧急状态或他人未经同意泄露后，**第三人再次公开** 的，视同首次公开行为，**不重复计算宽限期**。

## 5. 创造性常见陷阱（Judge agent 扫描清单）

这些是看似具备新颖性但实际不过三步法的常见情况：

### 陷阱 1：简单数值替换

**定义**：参数微调且无新技术效果

**示例**：
```
本方案: 温度 82°C
对比文件: 温度 80°C
区别: 数值
效果: 无差异 (数值范围重合)
结论: 无创造性 (常规参数优化)
```

### 陷阱 2：上下位概念替换

**定义**：用上位或下位概念替换对比文件的概念

**示例**：
```
本方案: 卷积神经网络 CNN
对比文件: 神经网络
区别: 具体化
结论: 无创造性 (下位概念是上位概念的常见实例)
```

### 陷阱 3：等同手段替换

**定义**：功能相同的已知手段互换

**示例**：
```
本方案: 电机驱动
对比文件: 液压驱动
区别: 驱动方式
效果: 同一目的 (产生动力)
结论: 无创造性 (等同替换)
```

### 陷阱 4：公知常识组合

**定义**：两个或多个已知模块简单拼接，无协同效应

**示例**：
```
本方案: 模块 A + 模块 B
对比文件 1: 模块 A
对比文件 2: 模块 B
区别: 组合
协同效应: 无 (A 和 B 各自独立工作)
结论: 无创造性 (简单组合)
```

### 陷阱 5：惯用手段

**定义**：领域内常见的实现方式

**示例**：
```
本方案: 软件架构中使用 MVC 模式
对比文件: 其他架构模式
区别: MVC
结论: 无创造性 (MVC 是软件领域惯用手段)
```

### 陷阱 6：参数优化

**定义**：通过常规实验可达到的参数调整

**示例**：
```
本方案: 神经网络学习率 0.001
对比文件: 神经网络学习率 0.01
区别: 数值
效果: 提升收敛速度
结论: 无创造性 (网格搜索可得)
```

## 6. 特征对比实务

对每篇命中文件，建立对比表：

```markdown
### 命中 #N (CN1234567A)

| 本方案特征 (权要拆分) | 对比文件公开 | 段落 | 差异 |
|---|---|---|---|
| 特征 A: <描述> | 是 | 0023 | — |
| 特征 B: <描述> | 否 | — | 区别特征 |
| 特征 C: <描述> | 部分 | 0045 | X vs Y, 等同需分析 |
```

**判定**：

- 所有特征都 "是" → 破坏新颖性
- 存在一个或多个 "否" → 具备新颖性，进入创造性分析
- 存在 "部分" → 分析是否 "等同"

## 7. Judge 决策矩阵

Judge agent 根据三步法结果 + 陷阱扫描给出综合判决：

| 新颖性 | 区别特征数 | 显而易见区别数 | 陷阱命中 | 综合判决 |
|:---:|:---:|:---:|:---:|---|
| 破坏 | — | — | — | 🔴 红灯 |
| 具备 | ≥ 2 | 0 | 0 | 🟢 绿灯 |
| 具备 | ≥ 2 | 0 | 1 | 🟢 绿灯（加警告） |
| 具备 | ≥ 1 | 0 | 0 | 🟢 绿灯 |
| 具备 | ≥ 1 | 0 | 1 | 🟡 黄灯 |
| 具备 | ≥ 1 | 1+ | — | 🟡 黄灯 |
| 具备 | 0 (全部等同) | — | — | 🟡 黄灯 |
| 具备 | — | 2+ | 2+ | 🔴 红灯（创造性严重缺失） |

## 8. 实务补充

### 不破坏新颖性的情形

即使公开了，在以下情况下不破坏新颖性：

1. **申请日前 6 个月** 符合 24 条四种情形之一
2. **不完整公开** —— 对比文件只披露部分特征
3. **不同技术方案** —— 虽使用相同术语但解决不同问题
4. **对比文件包含矛盾信息** —— 导致该特征无法直接毫无疑义确定

### 破坏新颖性但不破坏创造性的情形

只有**抵触申请**。其他所有破坏新颖性的情形都同时破坏创造性。

### 创造性"突出的实质性特点" vs "显著的进步"

- **突出的实质性特点** = 三步法判断显而易见性
- **显著的进步** = 技术效果是否显著

2023 指南合并了这两个判据，都用三步法。Judge 不需要单独判断"显著的进步"。
