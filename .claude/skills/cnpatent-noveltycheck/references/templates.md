# 模板集合 —— cnpatent-noveltycheck

> 所有 agent 使用的模板。Guide agent 生成操作卡片和回填模板时复用这些。Judge agent 写 5_*.md 时使用 verified_outline / adjustment / rejection 模板。

## 1. verified_outline schema（绿灯输出）

这是 `5_verified_outline.md` 的格式，**必须严格对齐 cnpatent Planner 的输入 schema**，因为 cnpatent Phase 0 会直接读这个文件作为 `01_outline.md` 输入。

```markdown
▍主旨四段式（全文骨架）
  ① 针对什么问题：[1-2 句浓缩核心痛点]
  ② 采取什么方法：[1-2 句浓缩核心技术方案]
  ③ 基于什么原理：[1-2 句浓缩关键技术机理]
  ④ 带来什么提升：[1-2 句浓缩可量化效果]

一、发明名称：[≤25 字, "一种...方法/系统/装置"]

二、技术领域：[1 句双层定位, "本发明涉及...技术领域, 尤其涉及..."]

三、背景技术要点：
   - 现有技术概况：[工程需求 + 现有技术介绍]
   - 任务场景：[目标领域下的具体任务]
   - 现有技术局限（编号列举, 与发明目的优势条目一一对应）：
     （1）[局限 1]
     （2）[局限 2]
     ...
     （N）[局限 N]
   - 本发明引出句：[收束到"需要解决的技术问题"]

四·发明目的要点：
   - 总起句：本发明旨在..., 本发明的优势体现在：
   - 优势条目（编号列举, 与背景局限、技术效果三方对应）：
     （1）[标题 1]：[做了什么 → 通过什么机理 → 带来什么] ← [源:...]
     （2）[标题 2]：[做了什么 → 通过什么机理 → 带来什么] ← [源:...]
     ...
     （N）[标题 N]
   - 综述收尾句

四·技术解决方案要点：
   - 开场（引用附图）：图 1 为本发明示意图, 图 2 为流程图, ...
   - 技术步骤（编号列举, 步骤数 M ≥ 优势条目数 N）：
     （1）[步骤标题, 功能名词短语]：[概要 + 机理]
     （2）[...]
     ...
     （M）[...]

四·技术效果要点：
   - 总起句
   - 效果条目（与优势条目核心内容相同、标题措辞角度不同）：
     （1）[标题 1, 动作完成体, 与优势 1 描述同一改进]：[带来什么 + 量化]
     （2）[标题 2, 动作完成体, 与优势 2 描述同一改进]：[带来什么 + 量化]
     ...
     （N）[标题 N]
   - 综述收尾句

五、预计附图清单：
   - 图 1：[描述]
   - 图 2：[描述]
   ...
   （通常 4-7 张）

六、具体实施方式·步骤拆分：
   - 步骤 1：[标题], 简/繁=[简|繁], 预计子步骤数 [N]
   - 步骤 2：...
   ...
   - Writer-C 负责步骤 1..K, Writer-D 负责步骤 K+1..N

七、术语锁定表：
   | 学术 / 外文术语 | 本专利统一使用的术语 | 为什么选这个 |
   |---|---|---|
   | [术语 A] | [统一术语 A] | [理由] |
   | [术语 B] | [统一术语 B] | [理由] |
   ... (8-15 个)

八、篇幅预算：
   - 三、背景技术：~1000-1600 字
   - 四·发明目的：~500-900 字
   - 四·技术解决方案：~900-1400 字
   - 四·技术效果：~700-1100 字
   - 六、具体实施方式：~1200-3500 字
   - 全文目标：4500-7500 字

九、查新验证元信息：
   | 字段 | 值 |
   |---|---|
   | 查新日期 | YYYY-MM-DD |
   | 使用数据库 | incoPat 命令行, incoPat 语义, 抵触申请检索, Scholar, arXiv, CNKI |
   | 命中总数 (去同族) | [数字] |
   | 详读篇数 | [数字] |
   | 最接近现有技术 | [命中号] [标题] |
   | 区别技术特征 | 1. ... 2. ... 3. ... |
   | Phase C 判定时间 | YYYY-MM-DD |
   | 新颖性风险 | 低 |
   | 创造性风险 | 低-中 |
   | 三步法结论 | 非显而易见 |
   | 建议 | 🟢 进入 cnpatent 写作流程 |
   
   novelty_verified: true
   novelty_check_date: YYYY-MM-DD
   judge_model: opus
   judge_version: cnpatent-noveltycheck v1.0
```

**硬约束**：
- 背景局限 N = 优势条目 N = 效果条目 N（数量相等）
- 优势条目（k）与效果条目（k）描述同一项改进，标题核心内容相同但措辞角度不同
- 技术方案步骤数 M ≥ N
- 全角编号 `（1）（2）...`
- 大纲里**不写公式**（公式只在 cnpatent Writer-C/D）
- 大纲里**不出现** "显著 / 极大 / 至关重要 / 颠覆性 / 巧妙地" 等禁用词

## 2. adjustment_suggestions 模板（黄灯输出）

```markdown
# 调整建议 [方案名]

## 判定

- **新颖性风险**: 低
- **创造性风险**: 中 / 高
- **最接近现有技术**: <命中号> <标题>

## 创造性问题分析

**区别技术特征**:
1. <特征 1>
2. <特征 2>
...

**识别的陷阱**:
- [陷阱类型]: <具体描述>
- [陷阱类型]: <具体描述>

**三步法问题所在**:

<说明哪一步过不了, 例如:
"区别特征 2 属于公知常识, 技术启示来源: 教科书《模式识别》第 5 章">

## 可行调整方向

### 方向 1: [标题]

- **做法**: 在现有方案基础上引入 <具体特征描述>
- **预期效果**: 解决 <新的技术问题>
- **风险**: 需重新验证该特征是否已被公开
- **适用场景**: <什么情况下选这个方向>

### 方向 2: [标题]

- **做法**: 强化 <具体效果维度> 的量化指标
- **需要补充**: <实验 / 对比数据>
- **适用场景**: ...

### 方向 3: [标题]

- **做法**: 从通用 <领域 X> 收窄到 <特定子领域>
- **代价**: 牺牲一定保护范围换取更强的创造性证据
- **适用场景**: ...

## 下一步

请选择一个调整方向, 更新输入后 **手动重跑 Phase A**.

告诉 Claude: "按方向 N 重新生成大纲", Claude 会重启本 skill 从 Phase A 开始.

**注意**: 本 Judge 不会自动重跑 Phase A. 这是设计约束, 用于防止 infinite revision loop 和 AI 过度修改方案.
```

## 3. rejection_report 模板（红灯输出）

```markdown
# 拒绝报告 [方案名]

## 判定

- **新颖性风险**: 中 / 高
- **关键破坏命中**: #N (专利号)
- **破坏依据**: 完整的特征对照 + 段落引用

## 破坏新颖性的命中详情

### 命中 #N: <标题>

- **专利号 / 论文 ID**: 
- **公开日**: (必须早于本方案申请日)
- **申请人 / 作者**: 
- **特征对照**:
  | 本方案特征 | 命中公开情况 | 依据段落 |
  |---|---|---|
  | 特征 A | 是 | 0023 |
  | 特征 B | 是 | 0045 |
  | 特征 C | 是 | 0067 |
- **结论**: 全部特征已被披露, 无区别特征

## 抵触申请破坏（如适用）

[如果红灯原因是抵触申请, 单独说明]

- **抵触申请号**: 
- **申请日**: (早于本方案)
- **公开日**: (等于或晚于本方案)
- **与本方案的技术重合**: <描述>

## 建议

- **不建议**基于当前技术方案申请专利
- **可选方向**:
  1. 放弃当前方向, 转向其他技术路线
  2. 大幅调整方案, 引入 <具体差异点> 后重新走 Phase A
  3. 请专利代理人做完整检索, 确认本 Judge 的判定
     (本 skill 是 AI 辅助初筛, 不能作为最终判决)

## 免责声明

本报告为 AI 辅助初筛结果. 判定基于用户在付费库提供的命中记录.
最终的新颖性和创造性判断必须由专利代理人通过完整检索分析得出,
并由 CNIPA 审查员在实质审查阶段做出.
```

## 5. 操作卡片模板库（Guide 生成时复用）

### 5.1 incoPat 命令行检索卡片

```markdown
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【T1 必查】incoPat
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
URL:     https://www.incopat.com/
登录:    <根据 user_profile auth 字段填>
入口:    首页 → 高级检索 或 命令行检索

字段代码:
  TIABC  - 标题+摘要+权要合并
  TI     - 标题
  AB     - 摘要
  CL     - 权利要求
  IPC    - IPC 分类
  AD     - 申请日
  PD     - 公开日
  PA     - 申请人

检索式 1 (按特征 A):
  TIABC=(<关键词块 1>) AND TIABC=(<关键词块 2>) 
  AND IPC=(<主 IPC> OR <次 IPC>) 
  AND AD<=<今天>

检索式 2 (按特征 B):
  TIABC=(<关键词块 3>) AND IPC=(<次分类>) 
  AND AD<=<今天>

检索式 3 (扩展维度):
  TIABC=(<关键词块 4>) 
  AND AD<=<今天>

操作步骤:
  1. 运行检索式 1 → 同族合并 → 按相关度排序
  2. 读 Top 50 标题, 筛出 Top 15 候选
  3. 对 Top 15 读摘要 + 权要 1, 填入记录表
  4. 重复检索式 2 和 3, 合并结果去重
  5. 对合并 Top 10 精读全文

预计耗时: 30-45 分钟
停止准则: 最后 20 篇命中无新的高相关度文献
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 5.2 incoPat 语义检索卡片

```markdown
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【T1 必查】incoPat 语义检索
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
URL:     https://www.incopat.com/
入口:    高级检索页面 → "语义检索" 标签

操作:
  1. 粘贴以下方案描述到语义检索框:

  <2-3 句自然语言方案描述>

  2. 运行, 按相关度排序
  3. 读 Top 30 摘要, 筛出 Top 10 精读
  4. 与命令行检索结果合并去重
  5. 填入记录表

Why:
  关键词检索会漏掉 "换皮" 专利 —— 同样的技术用不同术语表达.
  语义检索通过向量相似度召回, 弥补 Derwent DWPI 的缺失作用.

预计耗时: 10-15 分钟
停止准则: Top 30 内无新高相关度命中
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 5.3 抵触申请检索卡片

```markdown
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【特殊】抵触申请检索 (仅 incoPat 可做)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  抵触申请 = 他人先申请 + 在你申请后才公开的文件.

  免费库 (Google Patents / CNIPA pss) 无此类数据.

操作:
  1. incoPat 高级检索
  2. 申请日字段: [<今天 - 18 个月>, <今天>]

  4. 关键词: 核心特征块 (简化版, 只保留最核心的 2 组)
  5. 读 Top 20 候选
  6. 填入记录表的 "抵触申请候选" 部分

预计耗时: 5-10 分钟
停止准则: Top 20 读完即可
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```
